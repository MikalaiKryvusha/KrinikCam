# /pause — завершение рабочей сессии

Криник ставит работу на паузу. Выполни рутину завершения сессии **по порядку**.
Между шагами коротко пиши в чат, что делаешь. Не пропускай шаги. Если шаг
падает — остановись, сообщи Кринику, не продолжай вслепую.

Все пути относительно корня проекта
(`/Users/kryvusha/ai_sandbox/KrinikCam`).

## Шаг 1. Зафиксировать статус и планы в STATUS.md

Обнови `STATUS.md`:
- **Что сделано в этой сессии** — конкретно, с привязкой к багам/фичам и файлам.
- **Текущая позиция** — что работает, что в работе, где находимся.
- **Планы на следующую сессию** — чёткий чеклист «с чего продолжить» (команды, файлы,
  что проверить первым). Помни: новый сеанс ИИ-агента стартует с пустым контекстом —
  пиши так, чтобы можно было сразу включиться.
- Конвертируй относительные даты в абсолютные (сегодня — узнай из контекста/`date`).

Сверься с активными документами багов в `bugs/` и отрази их статус.

## Шаг 2. Актуализировать README.md

Открой `README.md` и приведи в соответствие с реальным состоянием:
статус фаз, рабочие фичи, инструкции. README двуязычный (EN/RU) — поддержи оба.
Не выдумывай — отражай то, что реально сделано и проверено.

## Шаг 3. Собрать README.pdf

```bash
node tools/readme-pdf.mjs
```

Создаёт `README.pdf` в корне (gitignored, первый запуск качает headless Chromium).
Если падает на отсутствии `md-to-pdf` — запусти `node tools/setup.mjs` один раз.

## Шаг 4. Собрать билд приложения

```bash
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
node tools/build.mjs --no-ui
```

> ВАЖНО: без `JAVA_HOME` на JBR system java_home может вернуть Java 25 и сломать сборку.
> Должно закончиться `BUILD SUCCESSFUL` и `✅ Build complete → app/build/outputs/apk/debug/`.
> Если билд падает — остановись и покажи ошибки (`... 2>&1 | grep "^e:"`), не коммить сломанное.

## Шаг 5. Коммит и пуш в GitHub

`tools/commit.mjs` делает всё разом: бампит build-номер в `version.json`,
`git add -A`, коммит с версионным штампом, пуш в `origin/main`.

```bash
node tools/commit.mjs "<тип>: <краткое описание сделанного за сессию>"
```

Стиль сообщения (из AGENT_GUIDE): `feat:` / `fix:` / `docs:` / `refactor:` / `ci:` + одна строка.
Пример: `node tools/commit.mjs "fix: Go Live стримит в YouTube + ландшафтное превью (Bug 02, 03)"`

Сообщение коммита завершай строкой:
```
Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>
```
(commit.mjs принимает многострочное сообщение — передавай через `$'...\n...'` если нужно.)

## Шаг 6. Итог в чат

Кратко отчитайся Кринику: что зафиксировано, что собрано, хэш коммита, что пушнуто,
и одной строкой — главный план на следующую сессию. На этом пауза.

## Заметки

- Если идёт активная RTMP-трансляция на устройстве — её останавливать НЕ обязательно для
  этой рутины (сборка и git к устройству не обращаются). Но если переустанавливаешь APK —
  стрим прервётся; для просто паузы переустановка не нужна.
- README.pdf и APK gitignored — `commit.mjs` их не закоммитит, это нормально.
- Если пуш отклонён (não-fast-forward) — сделай `git pull --rebase` и повтори пуш,
  затем сообщи Кринику о расхождении.
