## 4. Архитектура приложения

### 4.1 Схема потока данных (Data Flow)

```
[ USB Webcam ]  [ Internal Camera ]  [ Audio Sources ]
      │                  │                   │
      ▼                  ▼                   ▼
┌─────────────────────────────────────────────────┐
│              Source Manager                     │
│  (горячее подключение, инициализация, профили)  │
└─────────────────────┬───────────────────────────┘
                      │  Raw frames (YUV/MJPG/H264)
                      ▼
┌─────────────────────────────────────────────────┐
│           Video Processing Pipeline             │
│  AutoRegulation → Crop/Frame → Filters → Mix   │
└──────────┬──────────────────────────────────────┘
           │  Processed frames
           ▼
┌──────────────────────────────────────────────────┐
│              Codec Manager                       │
│  Сканирует доступные HW MediaCodec блоки         │
│                                                  │
│  ┌───────────────┐    ┌───────────────────┐     │
│  │  Encoder #1   │    │    Encoder #2     │     │
│  │  H.264 (HW)   │    │   HEVC (HW)      │     │
│  └───────┬───────┘    └────────┬──────────┘     │
└──────────┼─────────────────────┼────────────────┘
           │                     │
           ▼                     ▼
┌──────────────────┐   ┌──────────────────────────┐
│  RTMP → YouTube  │   │  RTMP/SRT → Instagram    │
└──────────────────┘   └──────────────────────────┘
           │                     │
           ▼                     ▼
┌─────────────────────────────────────────────────┐
│             Streaming Monitor                   │
│  Битрейт, дроп кадров, пинг, состояние потоков  │
└─────────────────────────────────────────────────┘
```

## 5. Описание модулей

### :feature:usb
- Обнаружение подключённых USB-устройств через Android UsbManager
- Запрос разрешения на доступ к USB
- Инициализация UVC камеры через AndroidUSBCamera
- Hot-plug: мониторинг подключения/отключения, автоматический запуск сканирования
- Выдача событий через Kotlin Flow: `UsbDeviceEvent.Connected`, `.Disconnected`, `.PermissionGranted`

### :feature:capture
- Source Manager: реестр всех активных источников (USB камеры, встроенная камера, микрофоны)
- Получение raw фреймов от UVC камеры и передача в Processing Pipeline
- Поддержка нескольких источников одновременно
- Интерфейс `VideoSource` и `AudioSource` для унификации

### :feature:codec
- Сканирование доступных аппаратных кодеков через `MediaCodecList`
- Определение поддерживаемых форматов, разрешений, FPS, битрейтов
- `CodecProfile` — набор параметров для одного кодека (кодек, разрешение, FPS, битрейт, GOP)
- Предупреждение при выборе CPU-кодирования (высокая нагрузка)
- Управление параллельными экземплярами `MediaCodec`

### :feature:streaming
- Абстракция `StreamingTarget` (RTMP / RTMPS / SRT / RTSP)
- `PlatformProfile` — профиль платформы (URL, ключ стрима, кодек, разрешение, протокол)
- `MultiStreamManager` — запуск N потоков одновременно, каждый на свой таргет
- Встроенные шаблоны: YouTube Live, Instagram Live, TikTok Live, Twitch, Kick, Custom RTMP
- Быстрое редактирование stream key без пересоздания профиля

### :feature:autoregulation
- Анализ кадра (гистограмма яркости, перео/недоэкспонирование) через OpenCV или встроенные средства
- Регулировочные профили: Balanced, Fast, DarkPriority, LightPriority, Manual
- Обратная связь с камерой: отправка UVC control-запросов (яркость, экспозиция, баланс белого, насыщенность)
- Работает в отдельном coroutine с настраиваемым интервалом (по умолчанию 500 мс)

### :feature:processing
- Видео: кроп по рамке платформы, масштабирование, наложение sources (PiP)
- GPU-фильтры: яркость, контрастность, насыщенность, температура, чёткость (GPUImage)
- Аудио: шумоподавление, AGC, эхоподавление, микширование нескольких источников, уровни каналов
- Фоновые медиа: воспроизведение MP3-файла или URL-потока как источник аудио

### :feature:overlay
- Плавающие on-screen кнопки и меню (Jetpack Compose Popup / Canvas)
- Быстрые регулировки: яркость, контраст (смещение поверх авторегулировки)
- Рамки кропа для каждой платформы (разные цвета), перетаскиваемые жестом
- Реакции: анимации сердечек, лайков, конфети (GIF/Lottie)
- Наклейки и видеооверлеи (MP4 с прозрачным фоном)

### :feature:monitoring
- HUD overlay (настраиваемый, сворачиваемый): битрейт по каждому потоку, дроп кадров, CPU/GPU, заряд, WiFi/LTE сигнал
- VU-метры аудиоканалов
- История событий (отключение камеры, потеря сети и т.д.)

### :feature:profiles
- `DeviceProfile` — конфиг под конкретное устройство (HW кодеки, USB hub, оптимальные настройки)
- `StreamProfile` — конфиг стрима (платформа, кодек, разрешение, FPS, битрейт)
- Экспорт/импорт профилей в JSON-файл
- Библиотека готовых профилей для популярных устройств

### :feature:settings
- Переключение языка (RU, EN, ES, PT, ZH, HI, BN, FR, DE, AR) в 2 тапа + пункт «Следовать системе»
  (ручной выбор языка в Settings — bug 46, план `plans/16`; EN/RU уже готовы plans/13)
- Проверка обновлений библиотек (GitHub Releases API, фоновый WorkManager)
- Ссылки на GitHub, README.pdf, README.md (открываемые тапом)
- Тема (светлая / тёмная / системная)

## 8. Инфраструктура и автоматизация

### 8.1 Node.js инструменты (`/tools`)

| Файл                   | Назначение                                                                          |
|------------------------|-------------------------------------------------------------------------------------|
| `tools/build.mjs`      | Сборка APK/AAB через Gradle; параметры: `--release`, `--debug`, `--abi`            |
| `tools/commit.mjs`     | git add + commit с автоповышением версии кода + push                               |
| `tools/release.mjs`    | Сборка Release APK + создание GitHub Release с Release Notes + загрузка артефактов |
| `tools/version.mjs`    | Чтение/запись текущей версии (SemVer) из `version.json`                            |
| `tools/translate.mjs`  | Утилита для удобного добавления/проверки переводов                                 |

### 8.2 GitHub Actions (`.github/workflows`)

| Файл                       | Триггер             | Действие                                             |
|----------------------------|---------------------|------------------------------------------------------|
| `build.yml`                | push / PR           | Компиляция + lint + unit tests                       |
| `release.yml`              | push tag `v*`       | Release build + загрузка APK в GitHub Releases       |

### 8.3 Версионирование

- Формат: `X.Y (build NNN)` → например, `1.0 (42)`
- `version.json` — единый источник правды:
  ```json
  { "major": 1, "minor": 0, "build": 42 }
  ```
- `tools/commit.mjs` — повышает `build` на каждый коммит
- `tools/release.mjs` — повышает `minor` (или `major` при флаге `--major`), сбрасывает `build`
