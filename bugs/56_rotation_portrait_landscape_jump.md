# Bug 56 — Прыжок видео+заглушки при переключении портрет↔ландшафт (превью)

> Заведён: 2026-07-18 (Криник, живой тест). Приоритет: НИЗКИЙ (отложено Криником — «оставить как есть»).
> Только при НАСТРОЙКЕ (поворот заблокирован во время эфира → в эфир/запись НЕ попадает).

## Симптом

При переключении угла холста ПОРТРЕТ↔ЛАНДШАФТ (0↔90, 0↔270, 90↔0, 270↔0) в превью на мгновение
(1-2 кадра) видео И текст заглушки «прыгают»: содержимое слоя сначала занимает неверное положение/
масштаб, затем скейлится/трансформируется до корректного. На переходах 0↔180 и 90↔270 (размер тот же)
прыжка НЕТ. Глобально всё работает корректно — дефект только в момент перехода.

## Root cause

Два независимых GL-пайплайна меняют размер НЕ одним и тем же кадром:
- **Композитор** (`CompositorVideoSource`) меняет размер выходного буфера encW×encH (портрет 1080×1920 ↔
  ландшафт 1920×1080) — `resizeCanvasKeepingCamera` (атомарно с поворотом).
- **Превью-GL RootEncoder** пересчитывает вьюпорт по своему encoderSize (`gl.setEncoderSize` +
  `AspectRatioMode.Adjust`).
Кадр перехода всегда рассинхронен у ОДНОГО из них (композитор новый размер / RootEncoder старый, или
наоборот) → RootEncoder скейлит выход неверным аспектом 1-2 кадра → «прыжок».

## Что пробовали (0.7 37-39)

1. Атомарность поворот+размер в композиторе (один GL-пост) — не убрало (проблема на стороне RootEncoder).
2. Колбэк `onFrameReady`: RootEncoder переключает вьюпорт ПОСЛЕ первого кадра нового размера
   (`resizeCanvasKeepingCamera(...) { gl.setEncoderSize... }`) — уменьшило, но не убрало (остаётся
   1-2 кадра рассинхрона scope.launch/render-thread).

## Возможный полный фикс (не делаем — отложено)

Превью на ФИКСИРОВАННОМ размере холста: композитор всегда отдаёт один размер, портрет/ландшафт —
леттербоксом ВНУТРИ фиксированного кадра → RootEncoder никогда не ресайзит вьюпорт → нет прыжка.
Истинный размер энкодера ставится только на go-live (поворот там заблокирован). Риск: рассинхрон
аспекта превью/эфира, регрессии в повороте — требует живой приёмки. Крупнее, чем выгода.
