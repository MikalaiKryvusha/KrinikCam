# Bug 43 — Ручной ввод битрейта: перевести на Мегабиты/с (писать «4», а не «4000»)

> Заведён: 2026-07-18 (ревью Криником фичи «профиль кодера»). Приоритет: НИЗКИЙ (UX-полировка).
> Реализуется в менеджере профилей кодера (bug 41).

## Симптом

В варианте «Своё» ручной ввод битрейта — в килобитах/с (кбит/с): чтобы задать 4 Мбит, надо писать
`4000`. Криник хочет вводить в **Мегабитах/с** — достаточно написать `4` (или `4.5`).

## Root cause

Поле `field_video_bitrate_kbps` и helper `resolveBitrate` в `StreamPlatformsOverlay.kt` работают в
кбит/с: `customKbps.toIntOrNull() * 1000`. Пресеты-чипы уже в Мбит («2.5/4.5/6/9») — а ручной ввод
рассогласован (кбит), что и путает.

## План фикса

> Реализуется в рамках **`plans/14_encoder_profile_manager.md`** (S3 — поле битрейта в менеджере
> профилей кодера, ввод в Мбит/с).

1. Поле ручного ввода — в Мбит/с, с дробями: строка `customMbps`, парсинг `toFloatOrNull()`.
   `videoBitrateBps = round(mbps * 1_000_000)`. Пол/потолок как есть (0.5 Мбит … потолок платформы).
2. Строку `field_video_bitrate_kbps` → `field_video_bitrate_mbps` = «Битрейт (Мбит/с)» / «Bitrate
   (Mbit/s)» (EN/RU). Клавиатура — decimal (точка/запятая).
3. Инициализация из существующего значения: `videoBitrateBps / 1_000_000f` в поле (напр. 4000000 → «4»).
4. **Приёмка:** ввести «4.5» → сохранённый профиль `videoBitrateBps == 4_500_000` (юнит на
   парсинг/резолв); UI-проверка, что поле показывает Мбит.

> Согласовать с bug 41: если ручной ввод переезжает в менеджер профилей кодера — правку делать там.
