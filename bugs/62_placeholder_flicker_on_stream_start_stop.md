# Bug 62 — Заглушка мигает на СТАРТЕ/СТОПЕ трансляции (слой селфи-камеры)

> Заведён: 2026-07-19 (Криник, живой тест). Устройство: Titan 1, `com.kriniks.kcam.debug` 0.8 (4)-debug.
> Приоритет: СРЕДНИЙ (косметика на переходах go-live / stop).

## Симптом

В слое «Источник видео — селфи-камера» (видео живое):
- **На старте трансляции** на миг мелькает standby-заглушка, затем сменяется видеопотоком.
- **На стопе трансляции** — аналогично, снова мельком заглушка.

Заглушка не должна появляться на переходах: видео есть и до, и после go-live/stop — мигание лишнее.

## Репро

1. Слой «Источник видео — селфи» с живым видео.
2. Go Live → **на миг мигает заглушка** → видео.
3. Stop → **снова мигает заглушка** → видео.

## Root cause — ГИПОТЕЗА (не подтверждено)

На go-live/stop пайплайн, вероятно, пере-цепляет превью/поверхность (`stopPreview` + preview-restore) и
на время teardown/reattach кадры источника не идут → композитор кратко показывает standby-fallback
(заглушку). Т.е. заглушка = индикатор «нет кадров прямо сейчас», а на переходе есть короткий gap.

Родственно bug 48 (churn превью-поверхности при старте записи/стрима, EGL) и bug 53 (мигание LIVE-бейджа
на тех же переходах).

## План (первый шаг)

Проследить последовательность на go-live/stop: где рвётся поток кадров источника (stopPreview /
пересоздание поверхности) и не показывать standby, если у слоя ЕСТЬ живой источник (держать последний
кадр / не переключаться на заглушку при коротком gap на переходе).

## Связи

- bug 48 (churn поверхности на стоп/старт), bug 53 (мигание LIVE-бейджа) — те же моменты переходов.
- bug 17 / bug 47 (standby-заглушка), bug 60 / bug 61 (та же заглушка — отсутствие видео и её искажение).

## ✅ ROOT CAUSE + ФИКС (2026-07-19, беспроводной ADB, Titan 1)

**Root cause (подтверждён кодом):** на go-live (`configureCaptureRotation` → `changeVideoSource`)
композитор пере-инитит GL → `CameraSlot` пересоздаётся с `hasEverHadFrame=false`, а
`wantStandby = !hasEverHadFrame || …` включал заглушку МГНОВЕННО, пока камера-продюсер переоткрывается
и не отдал первый кадр → вспышка заглушки. На стопе — аналогично (пере-цепление превью перезапускало
пайплайн).

**Фикс (2 части):**
1. `CompositorVideoSource` — `STANDBY_STARTUP_GRACE_MS = 2500` мс: свежесозданному слоту даём догнать
   первый кадр ПЕРЕД показом заглушки (`wantStandby = if (hasEverHadFrame) …HOLD else …GRACE`). Камера
   переоткрывается за ~0.5–1 с ≪ грейса → заглушка не всплывает.
2. Смежно [[63]]: на стопе больше НЕ пере-цепляем живое превью → композитор не рестартует на стопе →
   слот не пересоздаётся → нет вспышки на стопе.

**Приёмка (харнес):** запись экрана go-live→stop одиночной рабочей камеры, разложена на кадры (4 fps).
Видео присутствует в КАЖДОМ кадре через оба перехода — **ни одной вспышки заглушки** «KrinikCam».
(При 4 fps вспышка <250 мс теоретически могла бы проскочить — финальную приёмку «глазами в реальном
времени» делает Криник.)

## ➕ Продолжение (Криник, 2026-07-19): вместо заглушки — иногда мигает ЧЁРНЫЙ кадр

После фикса заглушка на старте больше не мигает, но **бывает** (интермиттентно) мелькает ЧЁРНЫЙ кадр.

**Root cause:** на go-live композитор реинитит GL (`changeVideoSource`) → снапшот слота ЧЁРНЫЙ, пока
камера-продюсер переоткрывается (~350–400 мс на Titan). `schedulePreviewRestoreAfterStream` пере-цеплял
превью, как только `gl.isRunning`, — то есть РАНЬШЕ первого живого кадра → превью показывало этот чёрный
кадр. Интермиттентность = гонка «re-attach vs первый кадр камеры».

**Фикс:** превью пере-цепляем ТОЛЬКО после первого живого кадра камеры. `CompositorVideoSource`
экспонирует `hasLiveCameraContent()` (флаг `anyLiveCameraFrame`, считается на GL-потоке);
`schedulePreviewRestoreAfterStream` ждёт его перед `startPreview(tv)` (кап 1200 мс). TextureView держит
последний кадр, пока превью отцеплено → чёрного не видно, цепляемся уже на живую картинку.

**Приёмка (харнес):** 4× go-live/stop под запись экрана; яркость ЦЕНТРА кадра (видео) = 109–119 на всех
331 кадрах — **0 провалов в чёрный**. Лог: `live-frame wait 350–400ms` на каждом старте (фикс сработал).

## Статус
🟢 ФИКС ВНЕСЁН + harness-проверка (заглушка И чёрный кадр). Обе части — заглушка (грейс) и чёрный кадр
(отложенный re-attach) — закрыты. Ждёт приёмки Криника (субсекундный транзиент — лучше видно в реале).
