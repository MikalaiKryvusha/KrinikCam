# Bug 44 — Режим каналов звука: выбор из ТРЁХ (Стерео / Моно / Объединённое стерео)

> Заведён: 2026-07-18 (ревью Криником фичи «профиль кодера»). Приоритет: СРЕДНИЙ.
> Реализуется в менеджере профилей кодера (bug 41). Развилка закрыта Криником 2026-07-18.

## Симптом

Сейчас в профиле — бинарный переключатель «Стерео» (стерео↔моно). Криник: **неправильно.** Нужен
**выбор из трёх вариантов**, чтобы покрыть все запросы платформ и стримеров:

1. **Стерео** — истинное стерео: левый и правый каналы независимы (как с источника). `channels=2`.
2. **Моно** — одноканальная дорожка. `channels=1`. Канал = даунмикс `(L+R)` (не «только левый»,
   чтобы не терять сигнал, пришедший в один канал).
3. **Объединённое стерео** — двухканальная дорожка, где ОБА канала несут одинаковый даунмикс `(L+R)`
   (`channels=2`, `L == R`). Для площадок, которые ТРЕБУЮТ стерео-дорожку, но у стримера моно-источник
   — звук одинаков в обоих ушах, но контейнер стерео.

По умолчанию — **Стерео**.

## Почему три, а не два

Разные площадки/сетапы: кому-то нужен честный стерео, кому-то реальный моно (1 канал, меньше битрейта),
кому-то — стерео-контейнер с моно-содержимым (объединённое). Один тумблер это не покрывает.

## Root cause

Модель: `audioStereo: Boolean = true` (двоичное). Нужна замена на перечисление режимов. Кодер:
`prepareAudio(sampleRate, isStereo, bitrate)` — только флаг стерео/моно, БЕЗ управления даунмиксом.
Нужно **сверить по байткоду RootEncoder 2.4.7 (AudioEncoder/MicrophoneManager)**, суммирует ли он L+R
при `isStereo=false` или берёт только левый — от этого зависит, нужен ли свой PCM-даунмикс.

## План фикса

> Реализуется в рамках **`plans/14_encoder_profile_manager.md`** (S1 enum `AudioChannelMode`, S2
> проводка + PCM-даунмикс, S3 селектор из трёх). Ниже — детали режимов.

1. **Модель:** вместо `audioStereo: Boolean` — enum `AudioChannelMode { STEREO, MONO, JOINED_STEREO }`
   (дефолт `STEREO`). Живёт на профиле кодера (bug 41). Дефолт повторяет прежнее поведение (стерео).
   Room-миграция кодер-профиля учитывает это поле.
2. **UI:** в менеджере профилей кодера — селектор из трёх (радиогруппа/сегмент): «Стерео», «Моно»,
   «Объединённое стерео». Строки EN/RU (`field_stereo` есть; +`field_mono` есть; +`field_joined_stereo`).
3. **Кодер + даунмикс:**
   - `STEREO` → `prepareAudio(sr, isStereo=true, br)`, каналы как есть.
   - `MONO` → `channels=1`; канал = `(L+R)` даунмикс (с нормализацией/клампом). Если RootEncoder сам
     не суммирует — вставить свой PCM-микс перед энкодером.
   - `JOINED_STEREO` → `prepareAudio(sr, isStereo=true, br)`, но оба канала заполняем даунмиксом `(L+R)`
     (`L=R`). Требует своей PCM-обработки буфера микрофона.
   - Точка вставки даунмикса: перехват PCM между `MicrophoneManager` и `AudioEncoder` (сверить API
     2.4.7 — есть ли колбэк/фильтр PCM; если нет — обёртка источника аудио).
4. **Приёмка (ffprobe + анализ каналов):** подать сигнал ТОЛЬКО в один канал источника, записать по
   очереди три профиля →
   - Стерео: `channels=2`, каналы РАЗНЫЕ (сигнал в одном).
   - Моно: `channels=1`, сигнал присутствует (не потерян).
   - Объединённое: `channels=2`, каналы ИДЕНТИЧНЫ и несут сумму (сверить `ffmpeg -af channelsplit`
     + сравнение/корреляция дорожек).

## Реализация (2026-07-18, plans/14)
Модель `AudioChannelMode {STEREO, MONO, JOINED_STEREO}` + селектор из трёх в менеджере профилей кодера
СДЕЛАНЫ; проводка `prepareAudioFor`: STEREO→isStereo=true, MONO→isStereo=false. **ОСТАТОК:**
JOINED_STEREO пока стерео-passthrough (ffprobe: channels=2, но оба канала НЕ являются суммой L+R) —
истинный даунмикс L+R→оба канала требует PCM-фильтра между MicrophoneManager и AudioEncoder (сверить
API RootEncoder 2.4.7). TODO помечен в `RtmpStreamer.prepareAudioFor`.
