# `bugs/` — дефекты, трудности, поломки

По документу на дефект: симптом, детерминированное репро, форензика, root cause / гипотезы, история
фикса, статус. Долговечный беклог багов самого агента — ничего не теряется, любой баг может подхватить
«с холода» будущая сессия. Один баг — один файл `NN_<name>.md`.

**Кринику (владельцу):** можешь завести баг простыми словами (что не так, как воспроизвести) — агент
приведёт его к канону. Смотри сюда, чтобы видеть известные дефекты и их статусы.

**ИИ-агенту:** наткнулся на дефект в ходе работы/теста — заведи документ по канону (навык
`/report-bug`; метод — `BUG_FIXING_FRAMEWORK.md`), даже на мелкий. Пока баг открыт — без тега `DONE`.
Починен **и проверен** — `git mv NN_x.md NN_DONE_x.md` + секция `## ✅ СТАТУС: DONE (дата)`. После
3 неудачных слепых попыток фикса — стоп, переключайся на исследование (`/bug-research`).
