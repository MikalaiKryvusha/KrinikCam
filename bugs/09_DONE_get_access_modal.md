# Bug 09 — USB-диалог «Дайте доступ к камере» вылезает при каждом переподключении

> Статус: **✅ ЗАКРЫТ** (2026-06-28, подтверждено Криником «с нуля»).
> Файлы: `app/src/main/AndroidManifest.xml`, `app/src/main/res/xml/device_filter.xml`.
>
> Итог: после фикса диалог НЕ показывается вообще — при запуске по `USB_DEVICE_ATTACHED` Android
> авто-выдаёт доступ к устройству (грант персистентен). Криник проверил с чистого старта: камера
> поднимается с живым превью, без диалога и без стопки. Навязчивость убрана полностью.
> Жалоба Криника: на горячую отключаю/подключаю камеру — диалог появляется снова и снова.
> Договаривались — показывать один раз за сессию (или вообще убрать навязчивость).

---

## Симптом

При каждом физическом переподключении USB-камеры всплывает системный диалог Android
«Разрешить приложению KrinikCam доступ к USB-устройству?». Очень навязчиво при hot-replug.

## Root cause

Это **системный USB-диалог** Android, а не наша in-app модалка. Поток:
`UsbViewModel` на `DeviceAttached` → `repository.requestPermission(device)` →
AUSBC `MultiCameraClient.requestPermission` → `USBMonitor.requestPermission`, который внутри:

```
if (usbManager.hasPermission(device)) processConnect(device)   // без диалога
else usbManager.requestPermission(device, permissionIntent)    // ПОКАЗ ДИАЛОГА
```

Android-разрешение на USB-устройство **привязано к конкретному подключению** и НЕ персистентно:
физический реплаг = новое подключение = `hasPermission` снова false = диалог снова.

Раньше у приложения **не было** `intent-filter` для `USB_DEVICE_ATTACHED`, поэтому в системном
диалоге даже не было галки «использовать по умолчанию» — Android не мог запомнить грант, и
спрашивал каждый раз.

## Фикс — персистентный грант через manifest intent-filter + device_filter

Регистрируем у `MainActivity` фильтр на attach UVC-устройства. Это включает в системном диалоге
галку **«Использовать по умолчанию для этого USB-устройства»**; как только она отмечена, Android
**автоматически выдаёт** доступ при всех последующих подключениях — `hasPermission`=true →
`USBMonitor` идёт по ветке без диалога. Диалог показывается максимум один раз.

`AndroidManifest.xml` (внутри `<activity .MainActivity>`):
```xml
<intent-filter>
    <action android:name="android.hardware.usb.action.USB_DEVICE_ATTACHED" />
</intent-filter>
<meta-data
    android:name="android.hardware.usb.action.USB_DEVICE_ATTACHED"
    android:resource="@xml/device_filter" />
```

`res/xml/device_filter.xml` — матч UVC-вебок на уровне дескриптора устройства:
```xml
<usb-device class="239" subclass="2" />  <!-- Misc/IAD composite (EMEET Piko+ и большинство вебок) -->
<usb-device class="14" />                <!-- Video class на уровне устройства -->
```
Класс EMEET подтверждён через `adb shell dumpsys usb`: device `class=239 subclass=2`,
интерфейсы `class=14` (Video). Фильтр игнорирует хабы/ethernet (Realtek class 0/255).

Логику репозитория НЕ трогали: `USBMonitor.requestPermission` уже сам гейтит по `hasPermission`,
так что с персистентным грантом диалога не будет без единой строчки в Kotlin.

Бонус: `ui.mjs allow` уже умеет тикать галку «использовать по умолчанию» (`ALWAYS_TEXT`), так что
ИИ-агент может одобрить этот диалог один раз автономно.

## Фикс #2 — стопка из ~15 диалогов (регрессия от фикса #1)

После фикса #1 Криник увидел ~15 диалогов поверх друг друга. Диагноз через
`adb shell dumpsys activity activities`: в стеке **десятки экземпляров MainActivity**, каждый
запущен intent'ом `USB_DEVICE_ATTACHED`. Причина: у активности был дефолтный
`launchMode="standard"` → **каждый attach камеры порождал НОВЫЙ экземпляр** MainActivity. Каждый
экземпляр заново стартовал USB-мониторинг → re-enumerate → новый `requestPermission` → каскад
диалогов и лавина активностей (новый инстанс → монитор → attach → новый инстанс …).

Фикс: `android:launchMode="singleTask"` у MainActivity. Intent attach доставляется в
ЕДИНСТВЕННЫЙ экземпляр через `onNewIntent`, новые не создаются.

Проверено: после рестарта в стеке ровно 1 MainActivity; 3 повторных запуска подряд →
по-прежнему 1 экземпляр (стопка не растёт). ✅

## Проверка (нужен физический реплаг — руки Криника)

> На момент фикса приложение держало РАЗОВЫЙ грант с прошлого подключения, поэтому диалог сейчас
> не показывается и автоматически проверить персистентность нельзя — нужен реальный цикл реплага.

1. Отключить камеру, подключить снова → должен появиться диалог **с галкой «использовать по
   умолчанию»** (раньше галки не было). Отметить галку + Allow — или выполнить
   `node tools/ui.mjs allow` (сам тикнет галку и нажмёт разрешить).
2. Ещё раз отключить/подключить → **диалога быть НЕ должно** (авто-грант). ✅ = баг закрыт.
3. Если галки нет / диалог повторяется — проверить, что устройство матчится device_filter
   (`adb shell dumpsys usb` → class/subclass), при необходимости добавить match по VID/PID.

## Уроки

1. USB-permission в Android **непостоянно** и привязано к подключению. «Показать один раз» —
   это НЕ флаг в коде, а системная галка «use by default», которая работает ТОЛЬКО при наличии
   `USB_DEVICE_ATTACHED` intent-filter + device_filter в манифесте.
2. AUSBC `requestPermission` сам гейтит по `hasPermission` — отдельный guard в нашем коде не нужен.

## Исходная заметка Криника

> Модалка «Дайте доступ к камере» — задолбала! На горячую отключаю камеру — подключаю — вновь
> появляется. Договаривались, что будет показываться только однажды в текущей сессии. Можно вообще
> выпилить. Изучи, как работает, и уменьши навязчивость.
