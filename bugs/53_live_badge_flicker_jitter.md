# Bug 53 — Плашка статуса эфира/записи «пляшет»: мерцает битрейт, плашка сжимается-разжимается

> Заведён: 2026-07-18 (Криник, живое тестирование). Приоритет: НИЗКИЙ-СРЕДНИЙ (визуальная стабильность).

## Симптом

В плашке статуса записи/эфира («LIVE 12:34 · 4.2 Mbps») мерцает значение битрейта, а сама плашка
«туда-сюда пляшет» — сжимается и разжимается по ширине.

## Root cause

`MainScreen.kt:570 LiveBadge`:
1. **Скачет ширина текста** → плашка перевёрстывается каждую секунду. Строка = `LIVE` + `MM:SS` +
   ` · X.X Mbps`. Число символов меняется: битрейт `4.2`→`12.7` (разная ширина цифр, пропорциональный
   шрифт), секунды `0:09`→`0:10`, перекат минут. `Surface`/`Row` без фикс. ширины → пере-лейаут и сдвиг
   («пляшет»).
2. **Мерцание битрейта** — `bitrateKbps` берётся из `onNewBitrate` (сырой посекундный битрейт
   RootEncoder), он прыгает (3800/4200/3900…) → число визуально дёргается.

## План фикса

1. **Стабилизировать ширину:** табличные цифры (моноширинные) для чисел
   (`FontFeatureSettings("tnum")` / tabular figures) ИЛИ фиксированная минимальная ширина плашки; и
   якорить плашку по левому краю, чтобы смена ширины не двигала её центр. Тогда тик секунд/битрейта не
   перекраивает лейаут.
2. **Сгладить битрейт:** усреднять по нескольким сэмплам (скользящее среднее) и/или обновлять реже
   (напр. раз в 1–2 с) и округлять до целых Мбит на плашке (детализация в HUD, если понадобится).
   Цель — спокойное число без дёрганья.
3. **Приёмка:** на устройстве в эфире/записи — плашка стоит на месте, ширина не скачет при тике
   секунд и смене битрейта; значение битрейта плавное, не мерцает. (Видео до/после.)
