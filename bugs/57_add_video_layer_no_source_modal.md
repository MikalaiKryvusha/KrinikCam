# Bug 57 — «Добавить слой → Устройство захвата видео» не спрашивает источник модалкой

> Заведён: 2026-07-18 (Криник, приёмка мульти-источников 0.7 46). Приоритет: СРЕДНИЙ (UX).
> Родственные: idea 21 Фаза B (мульти-источники), bug 58/59.

## Симптом

При добавлении слоя «Устройство захвата видео» слой создаётся СРАЗУ с источником `None` (пустой,
показывает заглушку). Модалка «какой источник добавляем?» НЕ открывается. Криник: нужно сразу
открывать модалку выбора источника при добавлении слоя.

## Репро

1. Панель слоёв → «＋ Добавить слой» → «Устройство захвата видео».
2. Слой добавлен, но источник не спрошен — надо потом лезть в ⚙ и выбирать вручную.

## Root cause

`onAddVideoCapture` → `StreamViewModel.addVideoCaptureLayer()` → `RtmpStreamer.addVideoCaptureLayer()`
просто добавляет `Layer.VideoCapture(id, source=None)` без выбора источника. Пикер источника живёт
ОТДЕЛЬНО в модалке настроек слоя (⚙), не вызывается при добавлении.

## План фикса

1. При тапе «Устройство захвата видео» в меню «Добавить слой» — НЕ добавлять сразу, а открыть модалку
   выбора источника (тот же список `sourceOptions`, что в ⚙-настройках слоя).
2. По выбору источника — добавить слой С этим источником: `addVideoCaptureLayer(source)` (или добавить
   слой + сразу `setCameraLayerSource(newLayerId, source)`; вернуть id из addVideoCaptureLayer).
3. Альтернатива (проще): добавить слой с None и СРАЗУ открыть его ⚙-модалку настроек (settingsFor =
   новый слой) — переиспользует существующий пикер. Но чище — отдельная модалка «выбор источника при
   добавлении».
4. Приёмка: «Добавить → Устройство захвата видео» → модалка источников → выбрал вебку/фронталку →
   слой появился уже с этим фидом.
