# Bug 47 — Не появляется заглушка при отключении камеры «нагорячую»

> Заведён: 2026-07-18 (наблюдение Криника). Приоритет: СРЕДНИЙ. Родственные: bug 35 (детач не
> доходит до UI), bug 17 (качество standby-заглушки), plans/05 S7 (фейд-заглушка как состояние слоя).

## Симптом

Криник выдёргивает USB-камеру во время работы — **заглушка (standby-плейсхолдер KrinikCam) не
появляется.** Вместо ожидаемой заглушки «нет сигнала» слой остаётся чёрным (или держит последний
кадр), явного «камера отключена» пользователь не видит.

## Репро

1. Активная USB-камера как источник слоя, идёт превью/эфир.
2. Выдернуть камеру нагорячую.
3. Наблюдать слой: заглушки нет.

## Root cause (два слоя проблемы)

1. **Заглушка попросту не реализована.** `Layer.VideoCapture` с источником `None` рисует ЧЁРНУЮ базу —
   в коде прямым текстом: `Layer.kt:44` «Источник не выбран — слой рисует чёрную базу (позже — фейд-
   заглушку KrinikCam, plans/05 S7)». `VideoSource.None` в комментариях обещает «black frame /
   placeholder», но плейсхолдера как графики нет. То есть даже при корректном переходе в None юзер
   увидит чёрное, а не заглушку.
2. **Детач может не переводить слой в None.** По bug 35 путь отключения активной USB-камеры «мёртв»
   (`notifyUvcDisconnected` не вызывается) → источник залипает на мёртвом UVC, авто-фолбэк на None/
   виртуалку не срабатывает. Тогда не наступает даже условие для показа заглушки.

## План фикса

> Детальный план исполнения → **`plans/17_standby_placeholder_layer.md`** (связан с bug 35 / plans/11).
> Ниже — краткая версия.

1. **Сначала развести причины (замер):** при hot-detach проверить логом, ПЕРЕХОДИТ ли слой в
   `VideoSource.None` (или залипает на UVC — тогда это ветка bug 35). Определяет порядок работ.
2. **Реализовать заглушку-слой (plans/05 S7 / bug 17):** когда источник слоя = None (или камера
   отвалилась) — рисовать заглушку KrinikCam (лого/«Нет сигнала»/фейд), а не голый чёрный. Это
   состояние слоя, не отдельный тип (арх-решение plans/05). Композитор уже всегда жив (Phase 3) —
   добавить рендер плейсхолдера в базовый проход при None.
3. **Гарантировать переход в заглушку при отвале:** отвал активного источника переводит слой в
   «нет сигнала»-состояние → показывается заглушка. Зависит от корректного детач-пути (**bug 35 /
   plans/11**): чинить совместно или после него.
4. **Приёмка живьём:** выдернуть камеру → в течение ~секунды слой показывает заглушку (не чёрное, не
   фриз последнего кадра); воткнуть обратно → камера вернулась. Проверить и в превью, и в эфире
   (запись переживает отвал — регресс idea 10 не допустить).

## ✅ СТАТУС: DONE (2026-07-18, фокус-задача Криника)

**Открытие:** заглушка-графика УЖЕ СУЩЕСТВУЕТ (`StandbyPlaceholder` — «KrinikCam / Please stand by /…»),
показывалась ТОЛЬКО при `activeSource == None`. На hot-detach источник остаётся `UvcCamera` (bug 35), но
объект камеры (`usbState.activeCamera`) зануляется (`DeviceDetached`-обработчик, UsbViewModel:78-79) →
условие было false → чёрный холст без заглушки.

**Фикс (низкорисковый, UI-условие) — `MainScreen`:** заглушка показывается, когда живого кадра нет:
`activeSource is None || (activeSource is UvcCamera && usbState.activeCamera == null)`. Во время
эфира/записи подавляется (`!isActive`), чтобы не мигать при пере-открытии камеры.

**ПРОВЕРЕНО ЖИВЬЁМ (наблюдение):** добавлена харнес-команда `simulate-detach` (Idea 22) — эмулирует
отвал активной UVC через РЕПОЗИТОРИЙ (эмит `DeviceDetached` в SharedFlow → оба экземпляра UsbViewModel
обрабатывают, как реальный физический отвал; лог: событие обработано дважды). Скриншот ПОСЛЕ
simulate-detach: **появилась заглушка** «KrinikCam / Please stand by». Регресс: `select-source none` →
заглушка (ок), `select-source uvc` (камера готова) → камера, заглушки нет (ок).

**Остаток (не в этом баге):** полный детач-путь / фантомы источников — bug 35 (plans/11). Восстановление
при ре-подключении идёт штатным `DeviceAttached` (не затронут). Физический hot-plug Криником — опц. сверка.

## 🔁 Доработка (2026-07-18): переделана в per-слойную GL-заглушку — финал

Первая версия (Compose-оверлей поверх превью) была архитектурно неверной (жалоба Криника: заглушка
накрывала ВЕСЬ экран и НЕ двигалась со слоем; жила только в превью, не в эфире). Переделано по
указанию Криника «заглушка живёт ВНУТРИ слоя»:

- Заглушка рисуется композитором (`CompositorVideoSource`) В КВАДРАТЕ слоя-камеры (`StandbyImage`
  title+body текстуры), а НЕ Compose-оверлеем. Двигается/масштабируется со слоем, попадает в эфир/запись,
  появляется ТОЛЬКО в отвалившемся слое (остальная сцена цела).
- Hold замороженного кадра камеры (`enterCameraStandby`, до 10с) → кросс-фейд: кадр гаснет, остаётся
  только ПРОЗРАЧНЫЙ текст (без чёрной плашки) → возврат источника (`exitCameraStandby`) убирает заглушку.
- Пульс «дыхания» на розовом заголовке (как старое превью), подпись статична.
- Compose-оверлей `StandbyPlaceholder` из MainScreen удалён. Закрывает и bug 17 (качество freeze-кадра).

Подтверждено Криником живьём: «теперь правильно», «всё красиво, я вижу».
