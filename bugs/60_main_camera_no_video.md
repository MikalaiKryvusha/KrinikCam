# Bug 60 — Нет видео с ОСНОВНОЙ (задней) камеры в слое «Источник видео»

> Заведён: 2026-07-19 (Криник, живой тест). Устройство: Titan 1, `com.kriniks.kcam.debug` 0.8 (4)-debug.
> Приоритет: ВЫСОКИЙ — основная камера непригодна как источник.

## Симптом

В слое «Источник видео» при выборе **основной камеры** (задней встроенной) кадров нет — вместо
видео висит standby-заглушка «KrinikCam». При этом:
- **Селфи-камера** в таком же слое даёт видео нормально.
- Если добавить ВТОРОЙ слой «Источник видео» с основной камерой рядом с работающим селфи — в слое
  основной камеры видео так же нет (висит заглушка).

Т.е. молчит именно задняя/основная камера, независимо от того, одна она в сцене или вместе с селфи.

## Репро

1. Сцена, слой «Источник видео» → выбрать «Основная камера» → **нет видео, висит заглушка**.
2. Слой «Источник видео» → «Селфи-камера» → видео есть (для контраста).
3. Добавить второй слой «Источник видео» → «Основная камера» рядом с живым селфи → **в слое основной
   камеры видео нет**.

## Root cause — ГИПОТЕЗЫ (не подтверждено, нужен logcat камеры)

Слой создаётся (раз показывается standby-заглушка — значит слой есть, но `onFrame` от источника не
приходит). Кандидаты:
- **Задняя Camera2 не открывается / не отдаёт кадры** — неверный `cameraId` резолв (id 0 vs фактический
  логический id задней), `CameraDevice.StateCallback.onError` (ERROR_CAMERA_IN_USE / DISABLED /
  MAX_CAMERAS), либо сессия не стартует. Снять `adb logcat` в момент выбора → искать Camera2 open/error.
- **Задняя камера не заведена как источник в мультиисточниках** (idea 21 Фаза B / bug 50/57/58/59):
  фронталка и UVC подписаны, а задняя — нет, поэтому композитор не получает её поток.
- **Titan 1 specifics** — на этом планшете задний сенсор может иметь иной формат/ориентацию; выбранный
  выходной размер невалиден для него → сессия пустая.

## План (первый шаг — форензика, не вслепую)

1. `adb logcat` при выборе основной камеры: искать `CameraDevice`/`Camera2`/наш камера-провайдер —
   открылась ли, какой `cameraId`, есть ли onError / onDisconnected.
2. Сверить список камер девайса (`cameraIdList`, facing) с тем, что мапится под «Основную».

## Связи

- bug 50 (несколько слоёв камеры), bug 57/58/59 (мультиисточники), **idea 21 Фаза B** (камера как
  равноправный слой) — возможно, задняя камера не заведена в этот механизм.
- bug 32 (аспект селфи — соседняя камера-тема).
- Показ заглушки при отсутствии кадров — её искажение и мигание вынесены в **bug 61** и **bug 62**.

## ✅ ROOT CAUSE (OS-лимит) + ФИКС (2026-07-19, беспроводной ADB, Titan 1)

**Диагностика (файловый лог приложения):**
- Камеры: `id 0 = Основная`, `id 1 = Селфи`.
- **Основная В ОДИНОЧКУ открывается и стримит** (`Device camera 0 streaming into layer`) — видео есть.
- Ошибка вылезает ТОЛЬКО когда открыты ОБЕ: `Device camera 1 (селфи) streaming` ✓ + `Device camera 0
  (основная) error 2` = **`ERROR_MAX_CAMERAS_IN_USE`**.

**Root cause:** это **ограничение ОС/железа**, не баг кода. Titan 1 (как почти все планшеты/телефоны)
физически НЕ тянет фронталку и тыл одновременно — один камера-HAL/ISP. Кто открылся первым — работает,
второй получает MAX_CAMERAS_IN_USE. (Нюанс: Android 11+ `getConcurrentCameraIds()` — на некоторых
устройствах огранич. комбо фронт+тыл; отдельная крупная переделка, Криник решил не тянуть сейчас.)

**Что было НАШЕ (и починено, реш. Криника «честный статус + убрать трэш»):**
1. **Трэш:** под RTMP-ретраями/старт-стопами каждый пере-prepare переоткрывал ОБЕ камеры → cam0 падал по
   кругу (open→error→close) — вероятный кормилец краша [[63]].
2. **Молчание:** конфликт показывал просто заглушку, без объяснения.

**Фикс:**
- `DeviceCameraOpener` — статический реестр `openBuiltinIds`: пока одна встроенная открыта, вторую
  даже НЕ пытаемся открыть (нет MAX_CAMERAS_IN_USE, нет churn). Резерв синхронный → победитель
  детерминирован порядком сцены (дефолтный слой открывается первым).
- `onConflict` → снэкбар (строка `camera_conflict_builtin`, EN/RU): «Это устройство не может
  использовать две встроенные камеры сразу…».

**Приёмка (харнес):** селфи + добавленная основная → в логе `НЕ открываю (bug 60)`, **ноль `error 2`**;
на экране — селфи-видео + снэкбар с честным текстом; заглушка на слое основной (16:9, [[61]]).

## Статус
🟢 ФИКС ВНЕСЁН + harness-проверка. Основная в одиночку работала и раньше; конфликт фронт+тыл — OS-лимит,
теперь обрабатывается честно и без трэша. Ждёт приёмки Криника.
