# Bug 08 — Экран Settings не скроллится (контент не влезает в ландшафте)

> Статус: **✅ ИСПРАВЛЕН** (2026-06-28). Файлы: `SettingsScreen.kt`, `tools/ui.mjs`.
> Нашёл Криник: повернул девайс в ландшафт — содержимое Settings не вмещается, свайп не скроллит.

---

## Симптом

В ландшафте секции Settings (Streaming / Permissions / Debug / About) не помещаются в высоту
экрана, а свайп вверх ничего не прокручивает — нижние строки (About → KrinikCam, Author)
недоступны.

## Root cause

Корневой `Column` в `SettingsScreen` не имел вертикального скролла. Compose `Column` НЕ скроллится
сам по себе — он просто клиппит то, что не влезло. (Та же природа, что Bug 04.3, где поля формы
обрезались в ландшафте.) Пока секций было мало, в портрете всё помещалось; после добавления секции
Permissions (и в ландшафте, где высота меньше) контент стал вылезать за вьюпорт.

## Фикс

`SettingsScreen.kt` — добавлен вертикальный скролл корневому `Column`:

```kotlin
Column(
    modifier = Modifier
        .fillMaxSize()
        .padding(padding)
        .verticalScroll(rememberScrollState())   // ← Bug 08
        .padding(horizontal = 16.dp),
    ...
)
```

Плюс `Spacer(Modifier.height(16.dp))` в конце списка — нижний отступ, чтобы последняя строка не
липла к краю при докрутке. Импорты: `androidx.compose.foundation.verticalScroll`,
`androidx.compose.foundation.rememberScrollState`.

## Инструмент: команда `ui.mjs swipe` (по просьбе Криника)

Чтобы ИИ-агент мог тестировать скролл, в `tools/ui.mjs` добавлена команда:

```
node tools/ui.mjs swipe <up|down|left|right> [fraction] [ms]
```

- Направление = направление движения пальца: `up` тянет контент вверх (показывает строки ниже),
  `down` — строки выше; `left`/`right` — горизонтально.
- Координаты берутся из ТЕКУЩЕГО прямоугольника экрана (через dump) → корректны в любой ориентации.
- `fraction` (доля экрана, дефолт 0.6) и `ms` (длительность жеста, дефолт 300) — опциональны.

## Проверка на устройстве (ландшафт 2560×1600)

1. Форс-ландшафт (`settings put system user_rotation 1`), открыть Settings → видны только
   STREAMING / PERMISSIONS / DEBUG, секция ABOUT/Author ниже фолда. ✅ (воспроизведён баг)
2. `ui.mjs swipe up 0.6 300` → дамп показывает ABOUT / KrinikCam / build-инфо / Author / Mikalai
   Kryvusha — нижние строки доехали. ✅ (скролл работает + команда swipe работает)
3. Авто-поворот возвращён устройству.

## Уроки

1. **Любой Compose-экран со списком секций должен иметь `verticalScroll`** (или быть `LazyColumn`) —
   иначе в ландшафте/на маленьких экранах контент молча обрезается. Проверять оба ориентации.
2. `ui.mjs` теперь умеет жесты прокрутки — тестируй скроллящиеся экраны через `swipe`.

> не работает. Ожидаемо: экран скролится свайпом. Понадобится: расширить инструмент автоматизации,
