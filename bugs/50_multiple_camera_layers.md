# Bug 50 — Нельзя добавить несколько слоёв «Устройство захвата видео»

> Заведён: 2026-07-18 (Криник, живое тестирование). Приоритет: СРЕДНИЙ. Один кластер с bug 16
> (камера — обычный слой) и idea 21 / plans/05 Фаза B (мультиинстанс в композиторе).

## Симптом

Криник хочет добавить в сцену НЕСКОЛЬКО слоёв «Устройство захвата видео» (напр. две камеры —
основная + PiP-вебка), но такой возможности нет — камера-слой один.

## Root cause

Камера сейчас «особенная»: она — базовый источник энкодера (низ пайплайна), а не равноправный слой.
Отсюда же bug 16 (у неё нет кнопки удаления). Мультиинстанс требует, чтобы камера стала ОБЫЧНЫМ слоем
поверх неявной чёрной базы, и композитор умел рисовать N камер-слоёв одновременно (каждый со своим
источником и трансформой). Арх-решение — `ideas/21_*`, фаза мультиинстанса — `plans/05` (Фаза B).

## План фикса

> Реализуется тем же арх-рефактором, что bug 16: камера = равноправный слой (idea 21). Затем
> **`plans/05` Фаза B — мультиинстанс** в композиторе.

1. Камера-слой становится обычным слоем сцены (не база) — общий шаг с bug 16.
2. Композитор рендерит несколько `Layer.VideoCapture` разом: каждый со своим `CaptureSource` и
   трансформой (z-order/scale/pos), потолок ~10 (арх-решение plans/05).
3. UI «Добавить слой» → «Устройство захвата видео» можно добавлять многократно; у каждого свой выбор
   источника (front/rear/uvc/virtual — свойство слоя) и свои настройки.
4. **Приёмка:** добавить 2 камеры-слоя (напр. встроенная + UVC), обе видны в композите (PiP), каждую
   можно двигать/масштабировать/удалять независимо; эфир/запись отдают композит с обеими; smoke зелёный.

## Связи

bug 16 (удаляемость камеры-слоя), idea 20 (выбор источника — уже свойство слоя), idea 21
(камера = слой), plans/05 Фаза B (мультиинстанс).

## Статус
🔴 ОТКРЫТ (средний). Требует арх-рефактора «камера = равноправный слой» (idea 21) + мультиинстанс
композитора (plans/05 Фаза B). Делается в связке с bug 16.
