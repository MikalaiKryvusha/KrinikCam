# Bug 12 — Превью теряет ручной поворот после поворота устройства туда-обратно

> Статус: **🔧 OPEN** (2026-06-29). Модуль: `:feature:usb` `UvcPreviewView` (Idea 06 матрица превью).
> Нашёл Криник.

## Симптом

1. Кнопкой выставляю угол поворота (напр. 90°) — превью поворачивается ✅.
2. Поворачиваю устройство, затем поворачиваю обратно.
3. Поворот «врёт»: картинка сбрасывается на исходные 0°, ХОТЯ переключатель угла всё ещё 90°.

То есть display-only матрица поворота превью (`UvcPreviewView.applyPreviewRotation`) теряется при
смене ориентации устройства (resize TextureView → рестарт GL-превью).

## Вероятная причина (гипотеза, проверить)

При повороте устройства срабатывает `onSurfaceTextureSizeChanged` → мы зовём
`applyPreviewRotation(tv, rotationDegrees)`, а ЗАТЕМ `startPreviewOnView` (рестарт GL,
`stream.startPreview(tv)`). Похоже, рестарт GL/привязка превью **сбрасывает матрицу
TextureView** (`setTransform`) ПОСЛЕ нашего вызова → матрица теряется. Плюс матрица могла считаться
по старым `tv.width/height` (до фактического resize).

## План фикса (focused)

1. Передавать в `applyPreviewRotation` ЯВНЫЕ w/h из колбэка (не читать `tv.width` в момент, когда
   они ещё старые).
2. Применять матрицу ПОСЛЕ рестарта GL (или повторно — напр. в `onSurfaceTextureUpdated` для первых
   кадров после resize), чтобы рестарт не затирал её.
3. `videoRotation` (StateFlow в VM) НЕ сбрасывается (activity `configChanges`=orientation, не
   пересоздаётся) — проверить, что матрица переустанавливается из актуального значения.

## Проверка (теперь автономно!)

Через виртуальную камеру (Idea 09) + `ui.mjs orient` (dev-тумблер «Вращение по ADB»): выставить 90°,
`orient landscape`→`orient portrait`, скриншот — круг/паттерн должны остаться повёрнутыми (90°), не
сброситься в 0°. Сверять по форме (круг круглый, ориентация TOP).
