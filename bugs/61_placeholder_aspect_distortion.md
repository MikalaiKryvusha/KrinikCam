# Bug 61 — Заглушка «KrinikCam» искажена по пропорциям (сжата по горизонтали)

> Заведён: 2026-07-19 (Криник, живой тест). Устройство: Titan 1, `com.kriniks.kcam.debug` 0.8 (4)-debug.
> Приоритет: СРЕДНИЙ (косметика, но заметно и бьёт по бренду).

## Симптом

Standby-заглушка «KrinikCam» (показывается, когда у слоя нет видео — см. bug 60) отрисована с
**неверным аспектом: сжата по горизонтали** (логотип/текст ýже натурального, вытянут по вертикали).
Пропорции картинки не сохраняются.

## Репро

1. Любой слой без кадров → показывается заглушка «KrinikCam» (например, слой основной камеры, bug 60).
2. Наблюдать: заглушка сплюснута по горизонтали, а не в натуральном соотношении сторон.

## Root cause — ГИПОТЕЗА (не подтверждено)

Битмап заглушки, вероятно, рисуется в композит/энкодер-буфер **растяжением под encW×encH** (fit-stretch
в 16:9) без сохранения аспекта — вместо letterbox / center-crop. При холсте с иным соотношением, чем у
исходной картинки заглушки, получаем сжатие по одной оси.

Родственно bug 32 (растянутый аспект селфи), но здесь это **статичная картинка заглушки**, а не
видеокадр, — путь отрисовки отдельный.

## План (первый шаг)

Найти отрисовку standby-заглушки (placeholder/standby в композиторе/GL), проверить, как её текстура
масштабируется в выходной буфер; добавить сохранение аспекта (letterbox или center-crop), а не stretch.

## Связи

- bug 32 (аспект селфи-камеры) — та же семья «неверный аспект».
- bug 17 / bug 47 (standby-заглушка) — тот же артефакт.
- bug 60 (нет видео с основной → там и видно искажение), bug 62 (мигание этой же заглушки).

## ✅ ROOT CAUSE + ФИКС (2026-07-19, беспроводной ADB, Titan 1)

**Root cause (подтверждён кодом + тестом):** заглушка-битмап уже 16:9 (`StandbyImage` 1920×1080), но
рисовалась через `finalM` слоя (`buildStandbyMatrix` строит `standbyM` из `finalM`), а `finalM` нёс
**аспект-фит под РОДНОЙ аспект камеры** (`layerMatrixOf`). У основной камеры сенсор 4:3 (1.324) →
фактор `a = 1.324 / 1.778 = 0.745` → квад сжимался по горизонтали → и видео (правильно, пилларбокс),
и заглушка (НЕПРАВИЛЬНО — она и так 16:9). Отсюда горизонтальное сжатие.

**Фикс (`CompositorVideoSource`):** `layerMatrixOf(layer, contentAspect)` — аспект передаёт вызывающий.
Видео камеры — в аспекте источника (пилларбокс, как было); **заглушка — в `SCENE_ASPECT` (16:9)**, т.е.
БЕЗ аспект-фита камеры. 16:9-заглушка → фактор 1, no-op → не сжимается.

**Приёмка (харнес, прицельно):** слот основной с аспектом **4:3** (`Device cam 0 … аспект 1.324`),
заглушка «KrinikCam» нарисована в тех же нормальных пропорциях, что и на 16:9-слое (ширина «KrinikCam»
одинаковая на обоих скринах). Без фикса при 4:3 была бы 0.745× — теперь нет.

## Статус
🟢 ФИКС ВНЕСЁН + harness-проверка (заглушка 16:9 при слоте 4:3, не сжата). Ждёт приёмки Криника.
