# Bug 46 — В Settings нет ручного выбора языка приложения (+ пункт «Следовать системе»)

> Заведён: 2026-07-18 (пожелание Криника). Приоритет: СРЕДНИЙ. Скорее фича, но заведено как задача
> по просьбе Криника «запиши баги».

## Симптом / хотелка

Сейчас язык UI определяется ТОЛЬКО системной локалью устройства (ресурсы `values/` EN-дефолт +
`values-ru/`, plans/13). Ручного переключателя языка в приложении нет. Криник хочет: на экране
**Settings — ручной выбор языка приложения**, среди опций — пункт **«Следовать языку системы»**.

## Контекст

- minSdk 33 (Android 13) → доступен НАТИВНЫЙ per-app language API: `LocaleManager.applicationLocales`
  (пустой `LocaleList()` = следовать системе). Не нужен costыль с `attachBaseContext`/пересозданием.
- Локализация уже сделана целиком (EN/RU, вкл. User Manual — plans/13). Не хватает только точки
  управления.
- `KrinikCamApp.attachBaseContext` существует (сейчас — только под логирование), локаль там не форсится.

## План фикса

> Детальный план исполнения → **`plans/16_app_language_selector.md`**. Ниже — краткая версия.

1. **Манифест + конфиг локалей:** `res/xml/locales_config.xml` со списком (en, ru) +
   `android:localeConfig="@xml/locales_config"` в `<application>`. Это включает системную интеграцию
   per-app языка (и пункт в системных настройках тоже).
2. **UI в Settings:** секция «Язык приложения» с выбором: **Следовать системе** (по умолчанию) ·
   English · Русский. Отрисовать как радиогруппу/выпадашку в `SettingsScreen`.
3. **Применение:** `getSystemService(LocaleManager::class.java).applicationLocales =`
   `LocaleList.forLanguageTags("ru")` (или `"en"`), а «Следовать системе» → `LocaleList.getEmptyLocaleList()`.
   (Кросс-версийно можно через `AppCompatDelegate.setApplicationLocales(LocaleListCompat…)`, если
   потянем appcompat; но нативного API достаточно при minSdk 33.) Система сама пересоздаёт Activity с
   новой локалью — Compose-экран мануала уже реагирует на смену локали (UserManualScreen — из ресурсов
   текущей локали).
4. **Персист:** выбор хранит система (applicationLocales персистентны) — свой DataStore не нужен;
   на старте просто читать текущее значение для отметки в UI.
5. **Приёмка:** сменить язык в Settings на Русский при системном English → весь UI и оверлеи
   по-русски (dump: 0 английских ключей в основных экранах), «Следовать системе» возвращает к
   системной локали; переживает перезапуск. Smoke зелёный.

## ✅ СТАТУС: DONE (2026-07-18, дневной цикл)

Реализовано и **ПРОВЕРЕНО ЖИВЬЁМ**:

- В Settings — секция «Язык» → «Язык приложения» с диалогом из трёх опций (Следовать системе /
  English / Русский), радиокнопки. Применение через `LocaleManager.applicationLocales` (Android 13+):
  пустой список = следовать системе, тег = принудительно. Система сама пересоздаёт Activity.
- **Приёмка (скриншоты):** секция показывает текущий язык; выбор English → ВЕСЬ UI на английском
  («Settings/HELP/User manual/LANGUAGE/App language/ABOUT»); выбор «Следовать системе» → вернулось на
  системный RU («Настройки/ЯЗЫК»). План — `plans/16`.
