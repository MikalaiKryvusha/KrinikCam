# Bug 04 — Модалка создания/редактирования платформы: 3 дефекта

> Статус: **✅ ЗАКРЫТ** (2026-06-28) — все 3 подбага исправлены и проверены на устройстве.
> Симптомы нашёл: Криник. Файл: `feature/streaming/.../ui/StreamPlatformsOverlay.kt`.

Три независимых дефекта в форме `ProfileEditDialog` (AlertDialog с полями платформы).

---

## 04.1 — Name перестаёт подставляться после первой смены платформы

**Симптом:** выбираешь платформу в выпадающем списке первый раз — её имя пишется в поле Name.
Меняешь платформу второй раз — Name больше не обновляется.

**Root cause:** условие автоподстановки сравнивало текущее имя с `initial` (исходным профилем):
```kotlin
if (name == initial.name || name == initial.platform.displayName) { name = p.displayName }
```
После 1-й смены `name` = "Twitch". На 2-й смене условие проверяет `name == initial.name`
("YouTube" — нет) и `name == initial.platform.displayName` ("YouTube" — нет) → не обновляет.
`initial` зафиксирован, поэтому проверка «живёт» только для исходного значения.

**Фикс:** считать имя «дефолтным» (можно перезаписывать), если оно пустое ИЛИ совпадает с
display-именем ЛЮБОЙ платформы из списка — тогда автоподстановка работает на каждой смене, а
кастомное имя пользователя (не совпадающее ни с одной платформой) не затирается:
```kotlin
val isDefaultName = name.isBlank() || StreamPlatform.entries.any { it.displayName == name }
if (isDefaultName) name = p.displayName
```

**Проверка:** YouTube → Twitch → Instagram. Name последовательно стал "Twitch", затем
"Instagram" (RTMP URL тоже обновился). ✅

## 04.2 — Тап мимо выпадающего списка закрывает всю форму

**Симптом:** в открытой форме открываешь выпадающий список платформ и тапаешь снаружи —
закрывается И список, И сам модальный диалог формы.

**Root cause:** `AlertDialog` по умолчанию имеет `dismissOnClickOutside = true`. Когда открыт
`ExposedDropdownMenu`, тап мимо него закрывает popup списка, но тот же тап «протекает» на scrim
диалога → срабатывает `onDismissRequest` → закрывается вся форма.

**Фикс:** отключить закрытие диалога по тапу снаружи:
```kotlin
AlertDialog(
    properties = DialogProperties(dismissOnClickOutside = false),
    ...
)
```
Теперь outside-тап закрывает только сам dropdown (его собственный `onDismissRequest`), форма
остаётся. Закрытие формы — кнопками Cancel/Save (back-кнопка тоже работает). Бонус: нельзя
случайно потерять наполовину заполненную форму.

**Проверка:** открыл dropdown (Twitch/TikTok/Custom видны) → тап по scrim слева → dropdown
закрылся, форма осталась (заголовок + Cancel/Save на месте). ✅

## 04.3 — Поля Width / Height / FPS пустые (без подписи и значения)

**Симптом:** три нижних поля формы (ширина, высота, FPS) выглядят как пустые рамки — ни подписи,
ни значения. Остальные поля (Platform/Name/RTMP/Stream key) рендерятся нормально.

**Root cause:** layout, НЕ данные. Значения в стейте были корректные. В **ландшафтной**
ориентации `AlertDialog` ограничивает свою высоту; стек из 4 высоких полей + ряд Width/Height/FPS
не влезал, и нижний ряд сжимался до ~0 высоты → `OutlinedTextField` рисовал рамку, но лейбл и
значение клиппились в невидимость. (Не теснота по ширине — каждое поле ~376px.)

**Фикс:** сделать контент диалога прокручиваемым, чтобы каждое поле сохраняло натуральную высоту:
```kotlin
Column(
    verticalArrangement = Arrangement.spacedBy(10.dp),
    modifier = Modifier.verticalScroll(rememberScrollState()),
) { ... }
```

**Проверка:** после фикса поля рендерят подписи Width/Height/FPS и значения 1920 / 1080 / 30,
редактируются; в ландшафте до них можно доскроллить. ✅

## Уроки

1. **Автоподстановка по «дефолтности»:** чтобы понять, можно ли перезаписать поле, сравнивай с
   полным множеством дефолтов, а не с одним зафиксированным `initial` — иначе логика «умирает»
   после первого изменения.
2. **`AlertDialog` + `ExposedDropdownMenu`:** outside-тап popup’а долетает до scrim диалога.
   Если форма не должна закрываться случайно — `dismissOnClickOutside = false`.
3. **Высокие формы в `AlertDialog` ОБЯЗАТЕЛЬНО оборачивать в `verticalScroll`** — особенно с
   учётом ландшафта на планшете (основная ориентация KrinikCam), где высота диалога мала.
   «Пустое поле» — частый симптом вертикального клиппинга, а не отсутствия данных.
