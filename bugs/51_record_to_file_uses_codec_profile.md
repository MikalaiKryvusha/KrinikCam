# Bug 51 — Запись в файл должна кодироваться выбранным профилем кодера

> Заведён: 2026-07-18 (Криник, ревью фичи «профиль кодера»). Приоритет: СРЕДНИЙ. Ложится в plans/14.

## Симптом / требование

Криник: запись видео в файл должна ТОЖЕ иметь настройку профилем кодера. **Какой профиль кодера
подключён к записи в файл — таким кодеком (разрешением/битрейтом/звуком) и пишется файл.** Сейчас у
записи нет явной привязки к профилю кодера — она пишет тем, что передали в `startRecordToFile(profile)`
(в харнесе — сконструированный на лету, в UI-кнопке «Запись» — активный/дефолтный профиль платформы).

## Root cause

Запись (`RtmpStreamer.startRecordToFile`) использует общий с эфиром путь `configureCaptureRotation`, но
профиль ей подаётся неявно, и отдельной пользовательской настройки «профиль кодера для записи» нет.
После рефактора (plans/14) профили кодера станут отдельной сущностью — запись должна ЯВНО ссылаться на
выбранный профиль кодера.

## План фикса

> Реализуется в рамках **`plans/14_encoder_profile_manager.md`** — добавить резолв профиля кодера для
> записи (шаг S2/S3).

1. Дать записи в файл собственный выбор профиля кодера (настройка «Профиль кодера для записи»), по
   умолчанию — активный/тот же, что у эфира. Хранить (DataStore/поле).
2. `startRecordToFile` резолвит выбранный `EncoderProfile` и конфигурит энкодер по нему (кодек,
   разрешение, битрейт, звук) — как эфир.
3. **Приёмка:** выбрать для записи профиль (напр. H.265/48k/mono), записать → ffprobe файла совпадает
   с профилем; сменить профиль → следующая запись пишется новым кодеком. (Метод — как приёмка plans/14.)
