# Interview #003 — Phase 2: RTMP от USB, мультистрим, UI-фиксы

**Дата:** 2026-06-27
**Тема:** Архитектура Phase 2 — рабочий RTMP с USB-камеры, Active Profile, мультистрим, UX-улучшения
**Статус:** ✅ Все вопросы закрыты

---

## Контекст

По результатам тестирования на устройстве (homework_before_phase2.md):
- **Критический баг**: Go Live крешится — `RtmpCamera1` (RootEncoder) пытается открыть Camera1/Camera2 API → конфликт с USB → нативный краш
- Портрет: видео растянуто по вертикали (нарушены пропорции)
- Ландшафт: видео боком на 90°
- FAB не закрывается тапом снаружи
- Dropdown платформ — плохой контраст
- Active Profile выглядит как заглушка

---

## Вопросы и ответы

### Q1: Что делает Active Profile в Phase 2?

| Вариант | Описание |
|---------|----------|
| **A (выбрано)** | Пресет = набор платформ. Active Profile — это комбо из нескольких платформ. Go Live → стрим одновременно на все платформы в профиле. Пример: Profile "YouTube+Twitch" = ключ YouTube + ключ Twitch |
| B | Просто указатель на одну активную платформу |
| C | Полный OBS-like preset (платформы + видео параметры + источник видео) |

> **Ответ:** A — Пресет = набор платформ

---

### Q2: Standby поведение при отключении USB-камеры

| Вариант | Описание |
|---------|----------|
| **A (выбрано)** | Буфер 5 сек + плавный fade. 5 сек ждём — вдруг камера вернётся. Если сигнал вернулся — без прерывания. Если нет — fade на "Please stand by". При повторном подключении — fade обратно на видео. USB permission диалог НЕ появляется для уже разрешённых устройств (фиксим это). |
| B | Мгновенная заглушка (как сейчас) |
| C | Настраиваемый таймаут в Settings (0–10 сек) |

> **Ответ:** A — 5 сек буфер + fade

---

### Q3: Кнопка поворота видео

| Вариант | Описание |
|---------|----------|
| A | Иконка поворота поверх превью (в углу) |
| B | В FAB-меню |
| **C (выбрано)** | On-screen dropdown с выбором угла: 0°, 90°, 180°, 270°. Выбрал — применилось — dropdown закрылся. Соотношение сторон меняется вместе с поворотом (16:9 ↔ 9:16) |

> **Ответ:** C — on-screen dropdown 0/90/180/270°

---

### Q4: Порядок реализации Phase 2

| Вариант | Описание |
|---------|----------|
| **A (выбрано)** | 1. RTMP crash fix (GenericStream для USB) → 2. UI-фиксы (FAB, контраст, ротация) → 3. Мультистрим |
| B | Сначала UI, потом RTMP |
| C | Всё вместе одним большим PR |

> **Ответ:** A — RTMP первым, это критический баг

## Итоговые решения Phase 2

| # | Тема | Решение |
|---|------|---------|
| Q1 | Active Profile | Пресет = набор платформ (несколько RTMP destinations). Go Live → стрим на все платформы в активном профиле одновременно |
| Q2 | Standby | 5 сек буфер → fade → "Please stand by". Обратно — fade → видео. USB permission для известных устройств — без диалога |
| Q3 | Ротация | On-screen dropdown: 0°/90°/180°/270°, применяется сразу, закрывается после выбора |
| Q4 | Порядок | 1. RTMP crash fix → 2. UI-фиксы → 3. Мультистрим |

### Полный список фиксов Phase 2 (по приоритету)

**P0 — критический:**
- [ ] RTMP crash: заменить `RtmpCamera1` → `GenericStream` с ручным push кадров из USB SurfaceTexture

**P1 — важные:**
- [ ] USB permission: не спрашивать повторно для уже разрешённых устройств (`PendingIntent` с нужными флагами)
- [ ] Standby: 5 сек буфер + fade-in/fade-out
- [ ] Ротация: on-screen dropdown 0°/90°/180°/270°
- [ ] FAB: закрывается тапом снаружи (dismiss on outside click)
- [ ] Dropdown платформ: контраст относительно фона диалога

**P2 — улучшения:**
- [ ] Active Profile = набор платформ (архитектура мультистрима)
- [ ] Multi-stream на несколько RTMP одновременно
- [ ] Кнопка "запросить разрешения снова" в Settings
- [ ] Camera2 fallback (телефонная камера)
