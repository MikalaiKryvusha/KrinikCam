## Контекст / что уже выяснил по коду

- Сейчас при отключении камеры во время стрима → **мгновенно** показывается заглушка
  (StandbyVideoSource, Phase 2 P2 — только что сделано и подтверждено).
- USB permission: `UsbViewModel` при **каждом** `DeviceAttached` безусловно зовёт
  `requestPermission()` → системный диалог появляется при каждом переподключении (это и есть
  старый Bug 3). Кэша/проверки `hasPermission` нет. В манифесте НЕТ intent-filter для USB.

### Технические факты (проверено)

1. **Диалог USB permission — НЕ жёсткое требование.** Android даёт штатный механизм:
   `<intent-filter android.hardware.usb.action.USB_DEVICE_ATTACHED>` + `@xml/device_filter`.
   Тогда при подключении устройства система один раз показывает диалог
   *«Открывать KrinikCam для этого USB-устройства? [✓] Использовать по умолчанию»*. После галочки
   **последующие подключения авто-разрешаются БЕЗ диалога**. Полностью убрать самый первый диалог
   нельзя (модель безопасности Android), но можно «один раз и больше никогда».
2. **Заморозку последнего кадра** можно сделать: захват кадра через `TextureView.getBitmap()` в
   момент отключения, затем рисовать его тем же механизмом, что и заглушку (StandbyVideoSource
   уже умеет рисовать произвольный bitmap в GL @ 5fps).
3. **Фейд «последний кадр → заглушка»** — просто (кроссфейд двух bitmap в draw-loop).
   **Фейд «заглушка → живая камера»** — сложнее: живая камера это отдельный GL-источник, нужен
   оверлей через `ImageObjectFilterRender` с анимацией alpha. Выполнимо, но это самый рискованный
   кусок.

## Предлагаемый план реализации (после ответов)

1. **Permission (Q3/Q4):** intent-filter + `device_filter.xml`; в `UsbViewModel` — проверка
   `hasPermission` перед `requestPermission` (не дёргать диалог, если право уже есть).
2. **Захват кадра:** в `onSurfaceDestroyed`/при detach — `TextureView.getBitmap()` → держим
   `lastFrameBitmap`.
3. **Состояние источника:** state-машина в RtmpStreamer (или новый `SourceTimeoutController`):
   `LIVE → FROZEN(5s) → STANDBY`, с мгновенной отменой при возврате камеры.
4. **Заморозка:** обобщить StandbyVideoSource → рисует любой bitmap; FROZEN рисует `lastFrameBitmap`.
5. **Фейд кадр→заглушка (500мс):** кроссфейд двух bitmap внутри draw-loop источника.
6. **(если Q2=A) Фейд заглушка→камера:** `ImageObjectFilterRender` оверлей + анимация alpha.
7. Сборка, установка, живой тест на устройстве (микро-дёрганье USB + полный таймаут).
