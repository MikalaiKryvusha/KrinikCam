## 1. Цель одной строкой

Прямое манипулирование слоями пальцами прямо на полноэкранном превью — как в OBS/фоторедакторе:
**перетащить** (1 палец), **масштабировать** (щипок), **вращать** (два пальца), выбрать/снять слой.
Жест меняет `LayerTransform` выбранного слоя вживую → композитор перерисовывает → видно сразу.

## 2. Что уже готово (переиспользуем, НЕ пишем заново)

- `LayerTransform(scale, cx, cy, alpha, rotation)` — нормализованная трансформа слоя (`scene/Layer.kt`).
- `RtmpStreamer.setLayerTransform(id, scale, cx, cy, alpha, rotation)` → `mutateScene` →
  `applySceneLayers` → `CompositorVideoSource.setLayers` (живой рендер). Есть StateFlow `scene`.
- Композитор рисует слой квадом с этой трансформой (`layerMatrixOf`), поворот аспект-корректный.
- CMD-харнес `set-transform` — уже дёргает тот же путь (для тестов без пальцев).
- Панель «Слои» (`StreamLayersOverlay`) — список слоёв, видимость, z-order, удаление.

**Вывод:** бэкенд трансформы полностью готов. План — про ЖЕСТ-UI поверх превью и выбор слоя.

## 3. Архитектура (слой за слоем)

### 3.1 Выбор активного слоя (selection)
Нужна сущность «какой слой сейчас редактируется жестами».
- `StreamViewModel.selectedLayerId: StateFlow<String?>` (null = ничего не выбрано).
- Выбор: **тап по слою** на превью (hit-test: какой видимый слой под точкой, сверху вниз по z) ИЛИ
  тап по строке в панели «Слои». Тап по пустому месту / вне рамки — снять выбор.
- Выбранный слой подсвечивается **рамкой-габаритом** (bounding box) с маркерами (по желанию, §8-Q3).

### 3.2 Жест-оверлей (Compose поверх превью, Layer 0.7)
Прозрачный `Box` на весь экран поверх превью-TextureView, ниже FAB/меню. Ловит жесты только когда
есть `selectedLayerId` (иначе прозрачен для тапов выбора).
- Compose `Modifier.pointerInput` + `detectTransformGestures` — даёт за кадр: `pan: Offset`
  (сдвиг в px), `zoom: Float` (множитель масштаба), `rotation: Float` (дельта угла в °), `centroid`.
- Отдельно `detectTapGestures` для выбора/снятия (когда слой не выбран).

### 3.3 Маппинг жеста → LayerTransform (КЛЮЧЕВОЕ, тут вся математика)
Жест в ПИКСЕЛЯХ TextureView надо перевести в нормализованную трансформу слоя. Тонкости:
1. **Леттербокс:** композит (16:9 или 9:16) вписан в TextureView с чёрными полями
   (AspectRatioMode.Adjust). Считаем видимый прямоугольник композита в TextureView (contentRect) —
   жесты мапим относительно НЕГО, не всего TextureView.
2. **Поворот холста:** при глобальном повороте (90/180/270) экранные оси не совпадают с осями
   сцены. Дельту `pan` перед применением к (cx,cy) надо **повернуть на −canvasRotation**, а `rotation`
   жеста — прибавлять как есть (слой крутится в координатах сцены). Иначе «тащу вправо, а слой едет
   вниз».
3. **Формулы (ландшафт, canvas=0):**
   - `cx += pan.x / contentRect.width` (и обратный знак по Y-экран→сцена как нужно);
   - `cy += pan.y / contentRect.height`;
   - `scale *= zoom`;
   - `rotation = normalize(rotation + gestureRotationDelta)`.
   Всё throttl-им (не чаще ~60 Гц) и клампим (§3.4).

### 3.4 Ограничения/снаппинг (чтобы не терять слой и удобно ставить)
- `scale ∈ [0.05, 4.0]` (не схлопнуть в точку, не раздуть бесконечно).
- Центр слоя держать в пределах, чтобы слой не улетел совсем (напр. `cx,cy ∈ [−0.1, 1.1]` или требовать
  ≥10% слоя на экране).
- **Снаппинг (по желанию, Q4):** угол к 0/90/180/270 в пределах ±5°; центр к 0.5/краям; лёгкая
  «магнитность». Мягкая тактильная отдача (haptic) на снап.

### 3.5 Применение и производительность
- Жест → пересчёт трансформы → `streamViewModel.setLayerTransform(id, …)`. Это дёшево (матрица в
  композиторе), но StateFlow-эмиссии дросселируем (conflate/sample ~16 мс), чтобы не спамить сцену.
- Во время СТРИМА трансформу менять МОЖНО (в отличие от глобального поворота — там смена разрешения).
  Проверить, что живой стрим не рвётся от частых setLayers (композитор просто меняет матрицы — должно
  быть безопасно; заложить тест).

## 4. Решения Криника (interview_007, 2026-07-06) — РАЗВИЛКИ ЗАКРЫТЫ

- **Q1 = A.** Выбор слоя: тап по слою прямо на превью И тап по строке в панели «Слои».
- **Q2 = B.** Только тонкая подсветка-рамка выбранного слоя; вся трансформа — жестами по площади
  слоя. Угловые маркеры — позже.
- **Q3 = C + УМНЫЙ снап.** Свободный угол + мягкий снап к 0/90/180/270. Снап **умный**: если видит,
  что слой «оторвали» (быстро/далеко от кратного угла) — даёт вращать свободно на малых углах без
  прилипания. Когда слой удерживается В ПРЕДЕЛАХ жеста около кратного 90° — показывать
  **вспомогательный ПУНКТИРНЫЙ штриховой маркер через центр слоя** (визуальный намёк на защёлкивание).
- **Q4 = A.** Снап включён сразу, с умным поведением из Q3 (мягкий, не мешает свободному вращению).
- **Q5.** Долгий тап по слою = **контекст-меню слоя (удалить / дублировать / на весь экран)** — делаем.
  (Двойной-тап-сброс НЕ делаем — Криник не выбрал.)
- **Q6 = A.** Один активный слой за раз.
- **Доп-требование Криника:** в ХАРНЕСЕ должна быть **логика автоматизации жестов**, чтобы ИИ-агент
  через инструменты тестирования мог сам минимально протестировать реализацию (не только руками).

## 5. Пошаговая реализация (инкрементально, каждый шаг проверяем)

- [ ] **S1. Выбор слоя.** `selectedLayerId` в VM; тап по панели «Слои» выбирает; подсветка строки.
      (Без превью-хиттеста пока.) Проверка: выбранный слой помечен.
- [ ] **S2. Drag (1 палец).** Жест-оверлей, `detectTransformGestures` → только `pan` → `cx,cy`.
      Маппинг с учётом contentRect (леттербокс), БЕЗ поворота холста пока (ландшафт). Проверка на
      харнесе: тащу камеру-PiP — двигается; запись подтверждает позицию.
- [ ] **S3. Pinch-scale + rotate.** добавить `zoom` → `scale`, `rotation` → `rotation` (аспект-корр.).
      Клампы §3.4. Проверка записью (портрет/пейзаж).
- [ ] **S4. Поворот холста в маппинге.** повернуть `pan`-дельту на −canvasRotation; проверить, что
      при глобальном повороте 90/270 драг «естественный». Проверка на всех 4 углах.
- [ ] **S5. Тап-выбор по превью (хиттест).** по точке найти верхний видимый слой (по его габариту с
      учётом трансформы+поворота холста); тап по пустому — снять. Рамка-подсветка на превью.
- [ ] **S6. УМНЫЙ снаппинг + haptic + пунктирный маркер (Q3/Q4).** Снап угла к 0/90/180/270:
      мягкий (защёлкивается в пределах ±5° при МЕДЛЕННОМ подведении), но при «отрыве» (быстрый/дальний
      жест) — свободное вращение без прилипания. Когда угол в зоне защёлкивания — рисовать
      **пунктирный штриховой маркер через центр слоя** (Compose Canvas оверлей). Лёгкая вибрация
      (haptic) в момент защёлкивания. Снап центра к серединам/краям кадра — аналогично мягко.
- [ ] **S7. Контекст-меню слоя (Q5).** Долгий тап по слою → меню: удалить / дублировать / на весь
      экран (сброс трансформы в full-frame). (Дублирование — новый id + копия слоя в сцене.)
- [ ] **S8. Харнес-автоматизация жестов (требование Криника).** Добавить CMD/`ui.mjs` команды, чтобы
      агент тестировал жесты без пальцев: `select-layer <id>`, `gesture-drag <id> <dx> <dy>`,
      `gesture-scale <id> <factor>`, `gesture-rotate <id> <deg>` — эмулируют жест → тот же путь, что
      пальцы (меняют LayerTransform). Реализация: либо broadcast → синтетическая трансформа (проще,
      минует Compose-жест), либо `input`-события в зону слоя (честнее, но хрупко). Дефолт — broadcast
      уровня НАМЕРЕНИЯ (как остальной харнес Idea 22). Прогон: `select-layer camera` →
      `gesture-drag camera 0.2 0.1` → запись → ffprobe/кадр (слой сместился).

**Милстоун:** слой (камера-PiP, картинка) двигается/масштабируется/вращается пальцами на превью,
корректно на всех поворотах холста, живёт в сцене; фундамент «OBS-лайт пальцами» (Phase 4).

## 6. Тестирование (харнес + железо)
- Автономно: `set-transform`/`select-layer` через CMD → запись → ffprobe/кадр (позиция/масштаб слоя).
- Вживую: пальцами на планшете — Криник (жесты руками харнесом эмулируются лишь частично).
- Регресс: трансформа во время стрима не рвёт RTMP; поворот холста + трансформа вместе.

## 7. Риски
- Маппинг координат при повёрнутом холсте — легко ошибиться в знаках/осях (S4 — самый аккуратный шаг).
- Спам StateFlow при жесте — дросселировать.
- Хиттест слоя с учётом поворота содержимого слоя (rotation) — геометрия OBB (oriented bbox); на
  первый заход можно axis-aligned приблизить.
