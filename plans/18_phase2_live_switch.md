# План 18 · Фаза 2 — Бесшовное переключение сцен В ЭФИРЕ

> Часть эпика [[idea 40]] / карта — [18_scene_profiles.md](18_scene_profiles.md). Стартует ПОСЛЕ Фазы 1.
> 📝 Набросок: детализируется при старте. Интервью не требуется (чисто техническая зона).

## Цель
Переключение активной сцены во время стрима — **без разрыва эфира, без чёрного кадра/мельтешения, 0 крашей**.
Ключевая OBS-фича: сцена «камера» → сцена «экран» → сцена «BRB» прямо в прямом эфире.

## Что шипается
- `setActiveScene(id)` работает и в состоянии `Live`: `applySceneLayers()` уже отдаёт слои композитору
  live (`CompositorVideoSource.setLayers`) — энкодер НЕ пересоздаём.
- Камеры переоткрывать ТОЛЬКО при РЕАЛЬНОЙ смене физ-источника: переиспользовать логику
  `sourceKey`/первичный-зеркало (bug 58) и грейс/отложенный re-attach (bug 62/#3). Слои с тем же
  устройством между сценами — не дёргать (шаринг фида цел).
- Разрешение/битрейт менять в эфире НЕЛЬЗЯ (ломает RTMP, [[idea 06]]) — сцены не меняют разрешение
  холста, только состав слоёв; проверить, что переключение не триггерит ресайз/поворот.

## Опора
`applySceneLayers` (live setLayers), `cameraLayerMirrors`/`layerSourceKeys`/`openedLayers` (bug 58),
`reopenDeadCameras`/грейс (bug 62/64), полигон `rtmp-server.mjs` + `go-live-rtmp`.

## Приёмка (харнес, полигон)
go-live → переключить сцену 5× → эфир не рвётся (сервер держит publish), без чёрного/заглушки-мельтешения,
0 крашей; стоп-стресс как в bug 63. Живая сверка Криника на реальном ключе (ДЗ-05).

## Риски
GL/камер-lifecycle (семья bug 58/62/63) — самая хрупкая зона; открытие/закрытие камер при смене сцены
должно идти строго через существующие гарды (`openedLayers`, primary/mirror). Аудио-дорожка непрерывна.
