# План 18 — Профили сцен (сохранение / рестарт / переключение / экспорт-импорт)

> Реализует [[idea 40]] (профили сцен, «мобильный OBS»). Высокоуровнево — шаг MASTER_PLAN «сцены».
> UI решён Криником: **FAB сцен рядом с FAB слоёв → радиальное меню быстрого управления сценами.**

## Цель

Сцена (набор слоёв с источниками/трансформами/z-order/видимостью/оверлеями) **сохраняется, переживает
рестарт, экспортируется/импортируется**; можно держать НЕСКОЛЬКО сцен и **бесшовно переключаться** между
ними, в т.ч. в эфире — из отдельного FAB-радиалки.

## Что уже есть (опора)

- `Scene(layers: List<Layer>)` иммутабельна; `Layer.VideoCapture(source: CaptureSource, transform, visible)`,
  `Layer.Image(bitmap, transform, visible)`; `LayerTransform(scale/cx/cy/alpha/rotation)`.
- Live-смена слоёв: `CompositorVideoSource.setLayers` (через `RtmpStreamer.applySceneLayers`) — переключение
  сцены без рестарта энкодера ложится сюда.
- Паттерн персиста + экспорт/импорт: `:data:profiles` (Room + DataStore, `buildExportJson`/`importProfilesFromJson`).
- Паттерн UI-радиалки: `ui/overlay/FloatingRadialMenu.kt`. Паттерн менеджера: `StreamPlatformsOverlay`/
  `EncoderProfilesOverlay`. FAB слоёв — внизу-слева (`MainScreen`), рядом ставим FAB сцен.

## Фазы

### Ф0 — доменная модель + сериализация (без UI, без персиста)
- `SceneProfile(id, name, layers: List<SceneLayerDto>, createdAt, updatedAt)`.
- `SceneLayerDto` — сериализуемый слепок слоя: тип (VideoCapture/Image), `CaptureSourceDto` (kind + id для
  Uvc/Builtin; Virtual/None), для Image — путь к файлу оверлея (НЕ bitmap), transform, visible, z-index.
- Мапперы `Scene ↔ SceneProfile` (в `:app` или мост, т.к. Image.bitmap живёт в UI-слое; загрузка bitmap из
  файла — `ImageOverlayLoader`).
- **Нюанс дефолтного слоя `camera`:** его источник сейчас идёт через `activeSource` (DeviceManager), а не
  scene `CaptureSource`. Для полного восстановления — класть источник дефолтного слоя в профиль тоже
  (устранить дуализм: сделать `camera`-слой равноправным по источнику из сцены).
- **Приёмка Ф0:** unit-тест round-trip `Scene → SceneProfile → JSON → SceneProfile → Scene` (кроме bitmap —
  сверяем путь).

### Ф1 — персист + автосохранение (переживание рестарта)
- Room-таблица `scene_profiles` + DAO (CRUD), фасад в `:data:profiles` (как StreamProfile).
- Активная сцена (id) — в DataStore. На старте приложения: восстановить активную сцену в `RtmpStreamer._scene`.
- **Автосейв текущей сцены:** любое `mutateScene` → debounce-запись слепка в «текущую» запись (чтобы работа
  не терялась даже без явного «сохранить»). Именованные профили — отдельные записи.
- **Оверлеи-картинки → файлы:** при добавлении оверлея копировать битмап в app storage
  (`filesDir/overlays/<uuid>.png`), в слое хранить путь; на восстановлении грузить из файла. Чистка сирот.
- **Приёмка Ф1:** собрать сцену из 3+ слоёв (камера + 2 оверлея, разные трансформы) → убить приложение →
  запустить → сцена восстановлена 1:1 (харнес: `ui.mjs`, сверка дампа/скриншота).

### Ф2 — FAB сцен + радиальное меню (решение Криника)
- Новый FAB «Сцены» рядом с FAB слоёв (внизу-слева). Тап → радиалка (компонент по образцу `FloatingRadialMenu`):
  - веер существующих сцен (тап = **переключить** активную сцену),
  - действия: «Новая сцена», «Дублировать», «Переименовать», «Удалить», «Экспорт/Импорт».
- Переключение сцены → `RtmpStreamer.setActiveScene(id)` → `_scene.value = …` → `applySceneLayers()` (live).
- **Приёмка Ф2:** создать 2 сцены, переключаться туда-обратно из радиалки → композит меняется корректно.

### Ф3 — бесшовное переключение В ЭФИРЕ
- Переключение активной сцены во время стрима: `setLayers` уже live; камеры переоткрывать ТОЛЬКО при
  реальной смене физ-источника (переиспользовать логику мультиисточников/`sourceKey`, bug 58, [[idea 21]]).
  Не рвать энкодер, не мигать (учесть фиксы bug 62/#3 — грейс/отложенный re-attach).
- **Приёмка Ф3:** go-live → переключить сцену 5× → эфир не рвётся, без чёрного/заглушки-мельтешения,
  0 крашей (харнес, стоп-стресс как в bug 63).

### Ф4 — экспорт / импорт набора сцен
- Экспорт: JSON (как платформы) + картинки. Варианты: (а) zip со сценами и ассетами, (б) JSON со ссылками
  на встроенные ассеты. Выбор — по развилке в [[idea 40]].
- Импорт: слить/добавить сцены (без затирания существующих, как `importProfilesFromJson`).
- **Приёмка Ф4:** экспорт набора → переустановка приложения (или второе устройство) → импорт → сцены
  (с оверлеями) восстановлены.

## Риски / внимание
- Оверлеи как bitmap-в-памяти → обязательно перевести на файлы, иначе сцена не сериализуется.
- Дуализм источника дефолтного слоя `camera` (activeSource vs scene) — развести до Ф1, иначе восстановление неполное.
- Переключение сцен в эфире — зона GL/камер-lifecycle (семья bug 58/62/63): аккуратно, с харнес-стрессом.
- Секретов в сцене нет (ключи — в профилях платформ), но при экспорте с ассетами следить за размером/приватностью.
