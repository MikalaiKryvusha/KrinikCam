> Заведён: 2026-07-18. Реализует: **bug 41** (хребет), **bug 42** (AV1), **bug 43** (битрейт в Мбит),

> Гейт: развилки закрыты Криником 2026-07-18 (см. ниже). Делаем ОДНИМ заходом — не коммитить старую
> структуру (кодер-поля на `StreamProfile`), сразу переносим в отдельную сущность.

## Замысел

Профиль кодера — **отдельная сущность** и **отдельный менеджер**. Платформа только ВЫБИРАЕТ профиль
кодера (ссылка по id), а как кодировать (разрешение/битрейт/fps/кодек/звук) — задаётся в менеджере
профилей кодера. Один профиль кодера переиспользуется многими платформами (нет дублирования параметров).

## Решения Криника (2026-07-18) — зафиксировано

- **Связь = живая ссылка по id, без снимков.** Правка профиля кодера задевает ВСЕ платформы на него.
- **Каналы звука — три режима** (дефолт Стерео): Стерео (L/R независимы) · Моно (1 канал = даунмикс
  L+R) · Объединённое стерео (2 канала, оба = даунмикс L+R, L=R). Подробности — bug 44.
- **AV1** добавляем в список кодеков (bug 42). **Битрейт «Своё» — в Мбит/с** (bug 43).

## Текущее состояние (что переносим)

Реализовано 2026-07-18 на `StreamProfile` (Room v3): `videoWidth/Height/Fps/BitrateBps`, `videoCodec`
(H264/H265), `adaptiveBitrate`, `audioBitrateBps/audioSampleRate/audioStereo`. UI-редактор — в
`StreamPlatformsOverlay.ProfileEditDialog`. Всё это уезжает в новую сущность/экран.

## Шаги

### S1 — Модель + Room (миграция v3→v4)
- Новая доменная сущность `EncoderProfile` (id, name, videoWidth, videoHeight, videoFps,
  videoBitrateBps, videoCodec, adaptiveBitrate, audioBitrateBps, audioSampleRate, **audioChannelMode**).
- `VideoCodec` += `AV1` (bug 42). Новый enum `AudioChannelMode { STEREO, MONO, JOINED_STEREO }`
  вместо `audioStereo: Boolean` (bug 44).
- Room: таблица `encoder_profiles` + DAO. `StreamProfile` теряет кодер-поля, получает
  `encoderProfileId: Long` (FK, живая ссылка).
- **Миграция v3→v4** (правило bug 37 — без destructive): создать `encoder_profiles`; для каждого
  существующего `stream_profiles` — создать профиль кодера из его текущих кодер-полей, проставить
  `encoderProfileId`; засеять 1 дефолтный профиль кодера. Кодер-колонки в `stream_profiles`
  оставить или дропнуть (SQLite: проще оставить «мёртвыми» либо пересоздать таблицу — решить в коде).
- `schemas/4.json` в git.

### S2 — Резолв + проводка энкодера
- При старте эфира/записи стример резолвит `StreamProfile.encoderProfileId → EncoderProfile` и
  конфигурит энкодер по нему (сейчас читает поля прямо у `StreamProfile`).
- `setVideoCodec` включая AV1 (`toPedro()` += AV1). Проверка аппаратной поддержки AV1 (`MediaCodecList`)
  → честный снэкбар при отсутствии (не молчать).
- `prepareAudio` по режиму каналов + PCM-даунмикс для MONO/JOINED (см. bug 44: сверить поведение
  RootEncoder, при необходимости свой микс L+R перед энкодером).
- Мультистрим: параметры энкодера — у профиля кодера ПЕРВОГО выхода (как сейчас `profiles.first()`).

### S3 — UI менеджера профилей кодера (сюда садятся bug 42/43/44)
- Отдельный экран/оверлей: список профилей кодера, CRUD (создать/править/удалить), импорт/экспорт JSON.
- Поля: разрешение (пресеты 16:9), FPS, **битрейт (Мбит/с, дробный ввод — bug 43)**, видеокодек
  (H.264/H.265/**AV1** + подсказки совместимости RTMP), адаптивный битрейт, битрейт звука, частота
  дискретизации, **режим каналов (Стерео/Моно/Объединённое стерео — bug 44)**.
- Точка входа: Settings и/или кнопка «Управление профилями кодера» из формы платформы.

### S4 — Форма платформы
- Убрать секции «Видео»/«Звук». Вместо них — одна выпадашка «Профиль кодера» (список профилей) +
  кнопка перехода в менеджер. Остаётся: имя, платформа, URL, ключ, вкл/выкл, выбор профиля кодера.

### S5 — Ссылочная целостность + импорт/экспорт
- Удаление профиля кодера, на который ссылаются платформы: запрет с предупреждением ИЛИ сброс этих
  платформ на дефолтный профиль (выбрать; не молча).
- Экспорт платформы тянет её профиль кодера в тот же файл (иначе импорт ссылается в пустоту); дедуп
  профилей кодера по содержимому при импорте. (Подтвердить деталь при реализации.)

### S6 — Приёмка
- **Миграция v3→v4 не теряет данные:** накатить новый билд поверх текущего (как idea 37/bug 37) —
  профили платформ живы, у каждого появился профиль кодера с прежними значениями.
- **ffprobe** записи через харнес по кодекам и режимам каналов: H.264/H.265/AV1 → соответствующий
  `codec_name`; Стерео/Моно/Объединённое → `channels` и идентичность/независимость дорожек (bug 44).
- Юниты: round-trip `EncoderProfile↔entity`, резолв FK, дефолты = прежнее поведение, дедуп импорта.
- Smoke зелёный, judge-проход.
