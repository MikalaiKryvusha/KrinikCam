## 1. Целевая модель (решения Криника из interview_006)

1. **Сцена** = набор слоёв, которые композитор пихает в логический холст **16:9**. Сцена «не знает»,
   как холст потом повёрнут, и не заботится об этом. У сцены есть верх.
2. **Глобальный поворот холста** — рантайм-тумблер **НАД сценами** (выше по иерархии): розовая кнопка
   вверху справа, углы 0/90/180/270. 90/270 → выход становится портретом **9:16** (1080×1920);
   0/180 → пейзаж 16:9. Поворот применяется ко ВСЕЙ композиции целиком: компоновка не меняется,
   картинка просто «легла на бок». Любая сцена, какую ни включи, ворочается этим поворотом.
3. **Слои** живут внутри сцены, к ним применима трансформация внутри сцены (аналогия — слои
   Photoshop): позиция/масштаб/альфа (есть) + **поворот содержимого слоя** (добавляем). Слой с
   текстом лёг набок при повороте холста → блогер сам выпрямляет его трансформой слоя.
4. **Физические камеры отдают СЫРОЙ поток** (16:9 как есть с сенсора), никакой самодеятельности с
   ориентацией устройства — поворотами занимается KrinikCam (= поворот холста + трансформа слоя).

      encoder-размер свапается на 90/270 (1080×1920) в `rotatedDims`. `configureCaptureRotation`

## 7. Известный REFINEMENT (не блокер Phase 3) — поворот переоткрывает камеру

**Находка (2026-07-06):** смена глобального поворота в превью зовёт `startPreview` →
`changeVideoSource(compositorSource)` → композитор рестартует → его OES-текстура камеры
пересоздаётся → `onCameraSurfaceReady(null→new)` → камера ЗАКРЫВАЕТСЯ и ПЕРЕОТКРЫВАЕТСЯ. Для быстрых
камер это кратковременный чёрный кадр; для флаки noname 2K-камеры (медленная негоциация, bug 25)
**после нескольких быстрых циклов поворота камера застревает в чёрном** (нужен перезапуск приложения).
При одиночной смене поворота и на быстрых камерах — работает.

**Почему так:** для 90/270 нужен ПОРТРЕТНЫЙ холст энкодера (1080×1920) → пересоздание GL-поверхности →
рестарт композитора (вместе с камерой). Причина не в матрице поворота (она верна — §6), а в связке
«смена размера холста ⇒ рестарт камеры».

**Кандидаты фикса (аккуратная задача, НЕ ночной слепой тык — риск GL/camera lifecycle):**
- (A) Сохранять OES-текстуру камеры и камеру-продюсера ЖИВЫМИ через смену размера холста композитора
  (пересоздавать только EGL window surface, не трогать камеру). Идеально, но сложнее.
- (B) Держать холст ПРЕВЬЮ всегда 1920×1080, поворот применять только матрицей (портрет = pillarbox
  в превью), а портретный размер энкодера ставить лишь на go-live/record. Проще, но превью портрета
  становится мелкой полосой (не 1:1 с выходом).
- Связано с bug 24 (остаточный чёрный при повороте) и bug 25 (медленная 2K-камера усугубляет).
