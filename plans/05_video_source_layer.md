# План 05 — Слой «источник видео» с выбором источника (Phase 4, idea 20)

> **Статус:** 🔧 В РАБОТЕ (2026-07-06). Источник: явное видение Криника (чат 2026-07-06, зафиксировано
> в `ideas/20_add_source_chooser.md`): «Никаких выбор по приоритету. Есть слои. Тип слоя — источник
> видео. Добавляешь в сцену. В настройках слоя выбираешь источник: селфи / фронталка / UVC (если видна)».
> Устройство: Titan 1 @ 192.168.50.187.
> **Требование Криника:** в плане ОБЯЗАТЕЛЬНЫ пункты тестирования инструментами автоматизации.

---

## 1. Целевая модель
- Камера-слой (`Layer.Camera`) имеет ЯВНО ВЫБРАННЫЙ источник: `Front` / `Rear` / `Uvc(deviceId)` / `None`.
- НЕТ авто-выбора по приоритету (уже убран: DeviceManager не лезет на встроенную; промежуточно
  авто-берёт UVC). Финально — источник задаёт ПОЛЬЗОВАТЕЛЬ в настройках слоя.
- Реестр доступных источников (`DeviceManager.uvcSources` + `phoneCameras`) уже есть — показываем как
  список выбора. UVC — только если реально подключена и видна приложением.

## 2. Scope
- **Фаза A (этот план):** у камера-слоя — выбор источника (Front/Rear/UVC/None) в панели «Слои».
  Один камера-слой. Заменяет авто-приоритет явным выбором.
- **Фаза B (позже, idea 20):** несколько видео-слоёв (2 камеры), кнопка «Добавить источник» с типами
  (источник видео / картинка / видео / текст). Требует нескольких OES-текстур в композиторе.

## 3. Шаги реализации (Фаза A)

- [ ] **S1. Модель источника слоя.** Ввести `enum LayerSource { FRONT, REAR, UVC, NONE }` (или
      `sealed` с `Uvc(deviceId)`). Добавить поле `source` в `Layer.Camera` (дефолт — авто/первый
      доступный UVC, иначе NONE). Пробросить в `Scene`/`RtmpStreamer`.
- [ ] **S2. Реестр доступных источников для UI.** `DeviceManager` уже отдаёт `uvcSources`/`phoneCameras`
      StateFlow. Добавить удобный `availableSources: StateFlow<List<VideoSource>>` (front, rear,
      подключённые UVC) для меню выбора.
- [ ] **S3. Применение выбора.** При смене `source` камера-слоя: `DeviceManager.selectVideoSource(...)`
      → `MainScreen` LaunchedEffect (уже есть) открывает нужный `CameraOpener` (Uvc/Device/Virtual).
      Убедиться, что смена источника корректно закрывает предыдущий и открывает новый (без churn/краша).
- [ ] **S4. UI выбора в панели «Слои».** В `StreamLayersOverlay` у камера-слоя — строка «Источник:
      <текущий>» с тапом → список доступных (Front/Rear/каждая UVC по имени/None). Выбор → S3.
- [ ] **S5. Харнес-команда** `select-source front|rear|uvc|none` (CMD-receiver + `ui.mjs cmd`) —
      чтобы я тестировал выбор источника автоматизацией (не только пальцами).

## 4. Тестирование инструментами автоматизации (ОБЯЗАТЕЛЬНО)

Для КАЖДОГО источника — прогон через харнес + визуальная/ffprobe-проверка:

- [ ] **T1. Выбор FRONT (селфи).** `node tools/ui.mjs cmd select-source front` → `sleep 3` →
      `node tools/ui.mjs screen` → кадр показывает ФРОНТАЛЬНУЮ камеру (лицо/потолок), не rear/UVC.
      Лог: `Active source updated: Front camera`.
- [ ] **T2. Выбор REAR (фронталка/тыл).** `cmd select-source rear` → screen → кадр с ТЫЛОВОЙ камеры.
      Лог: `Active source updated: Rear camera`.
- [ ] **T3. Выбор UVC (вебка).** `cmd select-source uvc` → screen → кадр с ВЕБКИ (широкий угол/то, на
      что смотрит вебка). Лог: `UVC connected` / `Active source updated: 2K USB Camera`.
- [ ] **T4. Выбор NONE.** `cmd select-source none` → screen → чёрная база + плашка «нет источника»,
      без краша (композитор рисует базу).
- [ ] **T5. Запись каждого источника:** `cmd stream-to-file on; cmd go-live; sleep 5; cmd stop` →
      `adb pull` → `ffprobe` (валидный MP4, 1920×1080) + извлечь кадр (`ffmpeg -ss 3 -frames:v 1`) →
      подтвердить, что записан ВЫБРАННЫЙ источник.
- [ ] **T6. Смена источника без краша:** быстро `front→rear→uvc→front` (циклом) → процесс жив
      (`adb shell pidof`), crash-буфер пуст (`logcat -b crash`). Проверка, что смена openers не роняет
      (риск нативного close-краша AUSBC, bug 28 — минимизировать переоткрытия).
- [ ] **T7. Регресс поворота+FBO:** при активном UVC — `set-rotation 90` → камера непрерывна, аспект
      верный (FBO-рефактор из plans/04 не сломан сменой источника).

## 5. Милстоун (Фаза A)
Пользователь (и я через `select-source`) выбирает источник камера-слоя: селфи / фронталка / UVC /
нет — явно, без приоритета. Каждый источник проверен записью+скрином. Фундамент под мультиисточники
(Фаза B) и жесты слоёв (plans/03).

## 6. Риск
- Смена openers (Uvc↔Device) закрывает камеру → нативный close-краш AUSBC (bug 28). Митигация:
  закрывать аккуратно, не частить; тест T6 это ловит.
- Если Фаза B (несколько OES в композиторе) — отдельный план, не сейчас.
