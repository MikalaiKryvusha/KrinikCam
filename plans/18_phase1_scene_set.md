# План 18 · Фаза 1 — Набор именованных сцен + менеджер в радиалке

> Часть эпика [[idea 40]] / карта — [18_scene_profiles.md](18_scene_profiles.md). Стартует ПОСЛЕ Фазы 0.
> 📝 Набросок: детализируется при старте (с учётом уроков Ф0). Перед стартом — **интервью (развилка 3)**.

## Цель
Держать НЕСКОЛЬКО именованных сцен и переключаться между ними (пока БЕЗ требования бесшовности в эфире —
это Фаза 2). FAB «Сцены» из Ф0 становится настоящим менеджером.

## Что шипается
- Room-таблица `scene_profiles(id, name, snapshotJson, createdAt, updatedAt)` + DAO (CRUD, `observeAll`) —
  по образцу `EncoderProfileEntity/Dao`. Бамп БД v4→v5 + `MIGRATION_4_5` (создать таблицу). Схема в git.
- Активная сцена — id в DataStore (как `active_profile_id`; ключ `active_scene_id`).
- Автосейв Ф0 переезжает с «одной текущей» на **активную запись** таблицы (debounce → upsert активной).
- FAB-радиалка (компонент по образцу `FloatingRadialMenu`): веер существующих сцен (тап = **переключить**
  активную) + действия «Новая / Дублировать / Переименовать / Удалить».
- Переключение: `setActiveScene(id)` → загрузить снапшот → `_scene.value = …` → `applySceneLayers()`.
  Дублирование оверлеев: копировать PNG-файлы под новые layerId (не шарить путь между сценами).

## Опора
`EncoderProfilesOverlay` (CRUD-менеджер), `FloatingRadialMenu` (радиалка), `ProfilesRepository`
(паттерн Room+DataStore+fallback), `SceneSnapshotDto/Mapper/ImageOverlayStore` из Ф0.

## Интервью (развилка 3, [[idea 40]])
Сколько сцен показывать веером радиалки? Нужен ли отдельный полный список-менеджер (как у платформ) при
многих сценах? Как выглядит переименование (инлайн/диалог)? Дефолтные имена новых сцен?

## Приёмка
Создать 2 сцены, переключаться туда-обратно из радиалки → композит меняется корректно; сцены переживают
рестарт (каждая своя); удаление активной снимает выбор (fallback на первую/дефолт, как у профилей кодера).

## Риски
Миграция Room (не потерять данные — destructive-фолбэка нет, забытая миграция роняет старт); копирование
оверлеев при дублировании; чистка файлов при удалении сцены.
