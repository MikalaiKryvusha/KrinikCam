# `plans/` — детальные планы работ

Детальные планы отдельных кусков работы: шагов генплана, конкретных фич, идей, багов, исследований,
процедур. Высокоуровневый маршрут — **`MASTER_PLAN.md`** (корень проекта); здесь лежат «приближенные»
планы, реализующие его шаги. Один план — один файл.

**Кринику (владельцу):** писать сюда не обязательно — планы обычно пишет агент. Можешь положить свой
план, если хочешь направить, КАК что-то делать. Читай их, чтобы видеть замысел агента до исполнения.

**ИИ-агенту:** перед нетривиальной работой напиши сюда короткий план и следуй ему. Новые планы нумеруй
(`NN_<name>.md`); исторические файлы фаз (`phase_*.md` и т.п.) оставлены с их именами — не переименовывай.
Завершённый и проверенный план получает тег `DONE` в имени (`git mv NN_x.md NN_DONE_x.md`) плюс секцию
статуса. Справочные документы (worklog'и, референсы — не закрываемая задача) тегом DONE не помечаются.
