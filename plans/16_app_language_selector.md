# План 16 — Ручной выбор языка приложения в Settings (+ «Следовать системе»)

> Заведён: 2026-07-18. Реализует: **bug 46**. Ветка Phase 9 MASTER_PLAN (локализация) + §5
> `:feature:settings` («Переключение языка в 2 тапа»). Тянем фичу вперёд по просьбе Криника.

## Замысел

Сейчас язык UI = только системная локаль (ресурсы EN + `values-ru/`, plans/13). Добавить в Settings
ручной выбор: **Следовать системе** (дефолт) · English · Русский. minSdk 33 → нативный per-app language
API (`LocaleManager`), без костыля `attachBaseContext`/ручного пересоздания.

## Шаги

### S1 — Конфиг локалей + манифест
- `res/xml/locales_config.xml` со списком поддерживаемых тегов (en, ru).
- `android:localeConfig="@xml/locales_config"` в `<application>` — включает системную интеграцию
  per-app языка (пункт появится и в системных настройках приложения).

### S2 — UI в Settings
- Секция «Язык приложения» в `SettingsScreen`: радиогруппа/выпадашка — Следовать системе / English /
  Русский. Строки EN/RU.

### S3 — Применение
- `getSystemService(LocaleManager::class.java).applicationLocales = LocaleList.forLanguageTags("ru")`
  (или `"en"`); «Следовать системе» → `LocaleList.getEmptyLocaleList()`. Система сама пересоздаёт
  Activity с новой локалью; экран мануала уже реагирует (ресурсы текущей локали).

### S4 — Состояние
- `applicationLocales` персистентны системой — свой DataStore не нужен; на старте читать текущее
  значение для отметки выбранного пункта в UI.

### S5 — Приёмка
- Системный EN → в Settings выбрать Русский → весь UI и оверлеи по-русски (dump: 0 английских ключей
  в основных экранах). «Следовать системе» возвращает к системной локали. Переживает перезапуск.
  Smoke зелёный.

## Статус
🔴 ОТКРЫТ. Инфраструктура локализации готова (plans/13); нужен конфиг + UI + вызов LocaleManager.
