# Idea 25 — Свой GL-композитор слоёв (финальная архитектура мульти-источников)

> ✅ ОКОНЧАТЕЛЬНОЕ архитектурное решение Криника (ночь 2026-06-30): **камера = ОДИН ИЗ СЛОЁВ (модель
> OBS Studio), НЕ базовый источник.** Путь «камера = `SurfaceFilterRender`-фильтр RootEncoder» отброшен
> (не доходил до энкодера / Surface abandoned, см. `bugs/18_*`). Делаем СВОЙ GL-композитор.

## Идея (KISS-вписывание в RootEncoder)

Не заменяем GlInterface RootEncoder. Делаем **`CompositorVideoSource`** — обычный RootEncoder
`VideoSource`, который САМ композитит все слои сцены и отдаёт ОДИН готовый кадр в SurfaceTexture
энкодера. Для RootEncoder это единственный источник → композит тривиально доходит и в энкодер, и в
превью. Камера при этом — лишь ОДИН из слоёв ВНУТРИ нашего композитора (удаляемый/переставляемый).

## Архитектура `CompositorVideoSource`

- `start(encoderSurfaceTexture)`: поднять EGL-контекст, рисующий в `Surface(encoderSurfaceTexture)`.
  Рендер-петля на своём потоке (~30 fps или по кадру камеры).
- Слои (z-order, из `Scene`):
  - **Камера** — OES external texture: своя `SurfaceTexture`, в неё рендерит камера (Camera2/USB через
    их продюсеры; виртуалка — рисует в неё через GL/Canvas). На кадр: `updateTexImage` → рисуем
    OES-квад шейдером (samplerExternalOES) с трансформ-матрицей SurfaceTexture.
  - **Картинка** — обычная 2D-текстура (загруженный bitmap), рисуем квад.
  - (далее: текст, видео, 2-я камера — те же квады/текстуры).
- На кадр: clear(чёрный) → для каждого ВИДИМОГО слоя снизу вверх рисуем его квад с его трансформом
  (позиция/масштаб/альфа) → `eglSwapBuffers`. Чёрная база = просто clear, когда слой не покрывает.
- Камера удаляется/прячется = не рисуем её квад (видно нижние слои/чёрный). Никаких костылей.

## Преимущества

- Полный контроль z-order/трансформа/видимости ЛЮБОГО слоя (включая камеру) — истинный OBS.
- Доходит до энкодера/записи/стрима (это базовый VideoSource).
- Переживает реинициализации GL RootEncoder лучше (наш EGL/текстуры под нашим контролем).
- Поворот/портрет — наша матрица на квадах (проще, чем боролись с RootEncoder).

## Риски / объём

Это БОЛЬШАЯ фича: EGL-контекст, шейдеры (OES + 2D), управление SurfaceTexture камеры, матрицы
трансформа, синхронизация потоков. Делаем ИНКРЕМЕНТАЛЬНО, проверяя на харнесе (запись в файл + ffprobe):
1. EGL + чёрный кадр в энкодер (база). Проверить: запись чёрная, корректные fps/размер.
2. + слой-картинка (2D-текстура) — проверить, что картинка в записи.
3. + камера-слой (OES): сперва ВИРТУАЛКА (рисует в нашу OES SurfaceTexture), потом Camera2/USB.
   Проверить: камера в записи (РЕШАЕТ блокер bug 18).
4. z-order/видимость/удаление камеры; трансформа (позиция/масштаб) — PiP.
5. Перенести `SceneCompositor`/`RtmpStreamer` на `CompositorVideoSource`; убрать путь
   `SurfaceFilterRender`-камеры.
6. Поворот/портрет (Bug 10) на матрицах композитора; сверить на записи и (позже) на YouTube.

- **Шаг 3 ✅ (ПРОРЫВ)** — слой-камера (OES external texture): компоновщик создаёт свою OES

  (камера-как-слой доходит до энкодера через наш композитор).
- **z-order (базовый) ✅** — камера (низ) + картинка (верх) композятся вместе, альфа корректна.

Файлы: `feature/streaming/.../gl/{Egl,GlQuadRenderer,CompositorVideoSource}.kt`. Интеграция: флаг
`useCompositor` в `RtmpStreamer` (мост), `cmd compositor on`, `onCameraSurfaceReady` → существующий
`CameraOpener` (вирт/USB/Camera2) открывает камеру в OES-поверхность композитора.

### Превью — РАБОТАЕТ (ложная тревога снята)

и в превью, и в энкодере: чёрная база + камера(OES) + картинка(2D), z-order, альфа.

### Шаг 4 ✅ ДОКАЗАН (z-order + видимость камеры)
Компоновщик рисует слои В ПОРЯДКЕ СЦЕНЫ (`CompositorLayer.Camera`/`.Image`, снизу вверх). Проверено
записью: камера НИЖЕ оверлея → оба видны (оверлей сверху); камера ВЫШЕ оверлея (`cmd layer-up camera`)
→ камера перекрывает оверлей; скрыть камеру (`cmd toggle-layer camera`) → чёрный+оверлеи. Камера —
полноценный равноправный слой (OBS). Новые tool-команды: `toggle-layer`/`layer-up`/`layer-down <id>`.

### Шаг 4b ✅ ДОКАЗАН — ТРАНСФОРМА СЛОЁВ (PiP), утро 2026-06-30
Каждый слой (камера И картинки равноправно) рисуется квадом с модельной матрицей позиции в clip-space:
`LayerTransform(scale, cx, cy, alpha)` — нормализовано к кадру (`scale` доля кадра, `cx,cy` центр в [0,1]
с экранным Y-флипом, `alpha` прозрачность). Дефолт = во весь кадр (без регрессий). Файлы: `scene/Layer.kt`
(+`LayerTransform`), `Scene.setTransform`, `GlQuadRenderer` (uniform `uPosMatrix` в VERTEX), `posMatrixOf`
в `CompositorVideoSource`, `RtmpStreamer.setLayerTransform` → repo/VM. Тонкая команда:
`ui.mjs cmd set-transform <id> <scale> <cx> <cy> [alpha]` (arg склеивается через запятую — иначе shell
на устройстве рвёт по пробелам; приёмник парсит по `[,\s]+`). Проверено записью+кадром:
  • камера PiP scale=0.4 в правый-нижний угол на чёрном — круг КРУГЛЫЙ (аспект сохранён), TOP upright;
  • классическая OBS-композиция: оверлей (рамка+бейдж) на весь кадр ФОНОМ + камера-PiP scale=0.35 в углу
    СВЕРХУ (`layer-up camera`) — z-order и трансформа работают ВМЕСТЕ. Истинный «лицо в углу».
