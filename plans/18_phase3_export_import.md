# План 18 · Фаза 3 — Экспорт / импорт набора сцен

> Часть эпика [[idea 40]] / карта — [18_scene_profiles.md](18_scene_profiles.md). Стартует ПОСЛЕ Фазы 2.
> 📝 Набросок: детализируется при старте. Перед стартом — **интервью (развилка 2)**.

## Цель
Сцену (или весь набор) выгрузить в файл и загрузить на другом устройстве / после переустановки — по
аналогии с экспортом профилей платформ/кодера ([[idea 01]], plans/12/14).

## Что шипается
- Экспорт: JSON снапшотов сцен (как `buildExportJson` у платформ) + картинки-оверлеи. Формат — по
  развилке 2 [[idea 40]]: **(а)** zip со сценами и ассетами, либо **(б)** JSON со встроенными ассетами
  (base64/ссылки). Учитывать размер/приватность.
- Импорт: слить/добавить сцены БЕЗ затирания существующих (как `importProfilesFromJson`; честный отчёт
  `ImportReport` — сколько добавлено/пропущено), распаковать оверлеи в `filesDir/overlays` под новые пути.
- UI: кнопки Экспорт/Импорт в менеджере сцен (SAF-пикер, как у платформ/кодера).

## Опора
`buildExportJson`/`importProfilesFromJson`/`ImportReport` (:data:profiles), SAF-пикер в
`StreamPlatformsOverlay`/`EncoderProfilesOverlay`, `SceneSnapshotDto`/`ImageOverlayStore` из Ф0/Ф1.

## Интервью (развилка 2, [[idea 40]])
Экспорт одной сцены или сразу всего набора? Формат с картинками (zip) — сейчас или отложить (сначала
JSON-only для сцен без оверлеев)? Куда класть файл (Downloads/share-sheet)?

## Приёмка
Экспорт набора → переустановка приложения (или второе устройство) → импорт → сцены с оверлеями
восстановлены; существующие сцены не затёрты; отчёт импорта честный.

## Риски
Размер пакета с картинками; приватность ассетов; совместимость версий формата (`SceneSnapshotDto.version`);
конфликты id при слиянии наборов.
