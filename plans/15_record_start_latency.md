# План 15 — Задержка старта записи в файл (режет начало фразы)

> Заведён: 2026-07-18. Реализует: **bug 45**. Ветка Phase 4 MASTER_PLAN (запись как UI-фича, idea 17).

## Замысел

Индикация «идёт запись» появляется РАНЬШЕ, чем реально пишутся кадры/звук → стример говорит в пустоту,
начало фразы теряется. Цель: (а) честная индикация по факту старта, (б) сократить сам прогрев.

## Шаги

### S1 — Замер (сначала измерить, потом чинить — BUG_FIXING)
- Залогировать Δ между `stream.startRecord(...)` и первым `RecordController.Status.RECORDING` /
  первым семплом в MediaMuxer. Разложить прогрев на составляющие: рестарт video-source в
  `configureCaptureRotation` (`changeVideoSource`), init `AudioRecord` (в логах был
  `dead IAudioRecord ... creating a new one`), прогрев GL (`schedulePreviewRestoreAfterStream` ждёт
  `gl.isRunning` до 5с). Понять главный вклад.

### S2 — Честная индикация (дешёвый фикс, точно входит)
- Переключать UI в «идёт запись» ТОЛЬКО по `RecordController.Status.RECORDING` (первый реальный
  семпл), до этого — состояние «подготовка…». Тогда юзер не начнёт говорить раньше времени.
- Затронуть: `RtmpStreamer.startRecordToFile` (сейчас `_state.value = Live()` СРАЗУ) + recordListener.

### S3 — Сократить прогрев
- Если главный вклад — рестарт video-source при записи (геометрия энкодера обычно = превью):
  пропускать `changeVideoSource`, когда размеры не изменились. Возможно держать энкодер/GL «тёплым».
  Аккуратно — не сломать поворот (configureCaptureRotation общий с эфиром).

### S4 — Приёмка
- Записать с хлопком/словом В МОМЕНТ старта → в файле начало на месте (хлопок слышен/виден).
- Δ старта до/после (должен упасть) и/или индикатор совпадает с реальным началом записи.
- Регресс не допустить: поворот записи, публикация в DCIM, idea 10/11/17 целы. Smoke зелёный.
