## Кто такой KrinikCam (для сравнения)

Android-приложение: USB-вебкамера через OTG → fullscreen превью → RTMP-стрим на YouTube/Twitch/etc.
Open-source (MIT), Kotlin/Compose, USB UVC через AndroidUSBCamera, RTMP через RootEncoder. Фишки:
радиальное FAB-меню, профили платформ, standby-кадр при отключении камеры, поворот видео.

## Категории конкурентов

1. **Прямые** — Android, USB/UVC-камера + лайв-стриминг (ближайшие к нам).
2. **Стриминг с телефона** — RTMP/SRT с камеры телефона (пересекаются по стримингу, не по USB).
3. **USB-камера вьюверы** — показывают UVC-камеру, но без стриминга (пересекаются по USB).

## Ключевые выводы для позиционирования (takeaways)

1. **Ниша KrinikCam защищена с двух сторон:** «стримеры» (Streamlabs/PRISM) слабы/отсутствуют по
   USB-вебке; «USB-вьюверы» (CameraFi viewer, отчасти Shenyao) не стримят или стримят утилитарно.
   Прямых «USB-вебка + стрим + стримерский UX» — мало (по сути CameraFi Live, отчасти Larix/Shenyao Pro).
2. **USB/UVC у конкурентов почти всегда «второй сорт»:** у Larix — премиум + experimental; у PRISM —
   часть мульти-кам; у Streamlabs — нет вовсе. У нас — **ядро продукта, бесплатно**.
3. **Деньги/watermark — общая боль:** watermark на free + платное снятие (CameraFi ~$8, Larix $9.99/мес,
   Streamlabs $27/мес). KrinikCam — **бесплатно, без watermark, open-source** → сильный SEO/маркетинг-аргумент.
4. **Стабильность USB — общая боль всех:** жалобы на чёрный экран (CameraFi Live), совместимость камер
   (Shenyao 2.84/5, CameraFi viewer 5.9/10), «экспериментально» (Larix). → наш акцент: **надёжное
   подключение вебки + standby-кадр** (поток не рвётся) = реальная ценность.
5. **Сложность vs простота:** Larix — порог входа высокий; PRISM/Streamlabs — раздутые комбайны. Наш
   месседж: **«включил вебку → пошёл в эфир», без возни** (idea 05).
6. **Куда расти (есть у них, нет у нас):** мультистрим (PRISM 6 каналов, Larix/CameraFi), SRT/HEVC/ABR
   (Larix), оверлеи/PIP/instant replay (CameraFi Live), алерты/темы (Streamlabs). → roadmap-ориентиры.
