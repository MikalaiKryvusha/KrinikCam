# Idea 39 — Разворачиваемый статус-виджет эфира (детальная телеметрия по платформам)

> Автор: Криник (чат, 2026-07-19). Расширяет статус-виджет из [[idea 37]] (LIVE / время / битрейт).
> Статус: ПРИНЯТА (видение Криника), ждёт реализации.

## Суть

Сейчас статус-виджет показывает свёрнутую сводку: **ЭФИР (LIVE)**, время эфира, суммарный битрейт.
Расширяем его до **разворачиваемого** по тапу.

- **Тап по свёрнутому виджету → разворачивается.** В развёрнутом виде показывает детально по КАЖДОЙ
  настроенной платформе мультистрима:
  - в какую платформу стрим **пошёл / подхватился** (Live), а в какую — **нет** (Failed/Reconnecting/
    Connecting) — то есть per-output статус;
  - статистика по выходу: **кодек**, **FPS**, **битрейт**;
  - **скорость интернет-соединения** устройства (аплинк);
  - **потерянные кадры**, **лаги/затыки сети** (congestion) и т.п.
- **Тап по развёрнутому виджету → сворачивается обратно** (тоггл).

## Зачем

Блогеру во время эфира важно видеть на одном экране, что реально уходит в YouTube, а что отвалилось от
Instagram, и почему (сеть/битрейт/дропы) — без копания в настройках. Свёрнутый вид не засоряет кадр,
развёрнутый — даёт полную картину по запросу.

## Что уже есть (опора для реализации)

Много данных уже собирается — [[idea 37]] (адаптивный битрейт + телеметрия):
- `OutputStatus` per-output (index, name, phase Live/Connecting/Reconnecting/Failed, bitrateKbps,
  attempt, reason, congested) — источник правды по каждой платформе (`RtmpStreamer.outputStates`,
  агрегат в `StreamState.Live.outputs`).
- Суммарный битрейт, длительность эфира, `hasCongestion()` per живой выход.
- Кодек/FPS — из активного `EncoderProfile` (plans/14).

Чего пока нет и надо добавить: измерение **аплинка устройства**, счётчик **потерянных кадров** (из
RootEncoder, если отдаёт), возможно — сглаженная история для мини-графиков.
