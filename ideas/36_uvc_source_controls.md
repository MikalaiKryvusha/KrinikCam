# Idea 36 — Управление параметрами USB-камеры (UVC-контролы источника)

> Источник: Криник (2026-07-07, дома с Piko+). Статус: ✅ ОДОБРЕНО (прямой запрос для MVP).
> Связано: plans/08_uvc_source_controls.md, idea 34 (модалка настроек слоя), plans/05 (слой-источник).

## Суть

Дать пользователю **крутить параметры, которые камера публично объявляет по USB (UVC-контролы)** —
яркость, контраст, усиление (gain), экспозиция, баланс белого, резкость и т.д. — прямо из KrinikCam.
Меню **динамическое**: заполняется по тем контролам, которые КОНКРЕТНАЯ камера декларирует наружу
(у разных камер разный набор).

Точка входа (по указанию Криника): **быстрая кнопка «Настройки источника» рядом с источником видео в
слое** (в модалке настроек слоя, idea 34) → открывает динамическое меню контролов с ползунками/тоглами,
крутящими параметры **вживую**.

## Зачем / боль

- **Прямо сейчас Piko+ жёстко лагает** (высокое усиление видеочипа), и настроить её через KrinikCam
  НЕВОЗМОЖНО — приходится мириться. С контролами можно сразу снизить gain/exposure и убрать лаг/шум/
  серый кадр.
- Стример должен управлять картинкой (экспозиция, ББ, яркость) — базовая потребность «мобильного OBS».
- Разные вебки — разные возможности; динамическое меню показывает ровно то, что камера умеет.

## Как вписывается в видение

- GOAL «мобильный OBS»: контроль над источником — ядро. Сейчас источник «как есть»; управление
  параметрами = следующий уровень.
- Переиспользует idea 34 (модалка настроек слоя): «Настройки источника» — раздел в ней для камера-слоя.
- MVP-приоритет Криника (наряду с мультистримом): «срочно, хотя бы в сыром виде».

## Набросок (KISS)

- AUSBC (libuvc) уже отдаёт get/set на весь набор: Brightness, Contrast, Gain, Exposure(+Mode),
  WhiteBalance, Saturation, Sharpness, Hue, Gamma, Zoom, Focus(+Auto), BacklightComp, PowerlineFrequency.
- Пробрасываем управление в UI: опросить какие контролы поддержаны → ползунок/тогл на каждый → set вживую.
- Детали реализации и этапы — plans/08.

## Открытые вопросы (не блокеры для сырого MVP)

- Как определить ПОДДЕРЖИВАЕМЫЕ контролы (UVC-дескрипторы vs проба get) — в MVP можно пробой get.
- Диапазоны (min/max) — если AUSBC не отдаёт, взять дефолтные/нормализованные (0–100), уточнить позже.
- Персист настроек на камеру между сессиями — на потом (MVP: крутим в рантайме).
