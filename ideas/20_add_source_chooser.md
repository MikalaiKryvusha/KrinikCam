# Idea 20 — Кнопка «Добавить источник» с выбором типа (а не «добавить картинку»)

> Источник: Криник, чат 2026-06-29 + УТОЧНЕНО 2026-07-06 (см. ниже).
> Статус: 🟡 запланировано (ключевая архитектура сцены/слоёв, Phase 4).

## ⭐ Уточнённое видение Криника (2026-07-06) — МОДЕЛЬ СЛОЯ «ИСТОЧНИК ВИДЕО»

Дословно: «Никаких "выбор по приоритету". Есть слои. Есть список типов слоёв. Один из типов —
**источник видео**. Добавляешь его в сцену. В настройках слоя выбираешь источник видео из доступных —
или **селфи**, или **фронталка**, или **UVC**, если подключена и видна приложением».

**Для архитектуры:**
- НЕТ авто-выбора источника по приоритету (текущий `DeviceManager.updateActiveSource` — стопгап).
- Слой типа «источник видео» имеет НАСТРОЙКУ: какой физический источник в него подаётся
  (front / rear / конкретная UVC-вебка). Список = реально доступные приложению.
- Пользователь ЯВНО выбирает источник для слоя. Можно несколько видео-слоёв (2 камеры).
- Криник: «удали из сцены слой встроенной камеры, добавь слой вебки и работай с ней» — дефолтный
  камера-слой не должен быть встроенной; при воткнутой вебке = вебка.

**Промежуточный фикс (2026-07-06):** `DeviceManager` больше НЕ авто-выбирает встроенную камеру
(приоритет UVC → виртуалка → НЕТ; встроенная — только по явному `selectPhoneCamera`). При запуске с
вебкой показывается вебка, а не встроенная.

**План модели «источник видео» (Phase 4):**
- [ ] `Layer.Camera` → `Layer.VideoSource(sourceId)` с полем выбранного источника (Front/Rear/Uvc(id)/None).
- [ ] Реестр ДОСТУПНЫХ источников (`DeviceManager`): front/rear/подключённые UVC — для меню выбора.
- [ ] UI настроек слоя: выбрать источник из доступных; смена = открыть/закрыть камеру (CameraOpener).
- [ ] «Добавить источник» → тип «Источник видео» создаёт такой слой. Несколько видео-слоёв — так же.

## Суть

В панели «Scene layers» кнопка добавления слоя сейчас называется «Add image overlay» — это
неправильно по смыслу. Кнопка должна быть **общей: «Добавить источник»**. По нажатию открывается
выбор КОНКРЕТНОГО типа источника, и картинка — лишь один из вариантов.

## Как должно быть

- Кнопка **«Добавить источник» / «Add source»** (одна, общая).
- Тап → меню/лист выбора типа источника. Состав по приоритету Q2 (interview_005):
  - 🖼 **Картинка (PNG/JPG)** — есть (SAF-пик, Idea 19 фаза 1).
  - 🎞 **Видео-оверлей** (в т.ч. анимации-стикеры с альфой) — `SurfaceFilterRender`.
  - 🔤 **Текст** — `TextObjectFilterRender`.
  - 📷 **Вторая камера** (встроенная фронталка) — Camera2 → `SurfaceFilterRender`.
  - 🟥 **Заглушка «ЗАГЛУШКА KrinikCam»** — особый fallback-источник (Q4) — фаза 2 дорожной карты.
  - 🌐 **Браузер** (`AndroidViewFilterRender` + WebView) — на потом.
  - 🖥 **Захват экрана** — минимальный приоритет, на будущее.
- По мере реализации типов — просто добавляются пункты в это меню. Каждый тип внутри ведёт к своему
  потоку настройки (для картинки — SAF-пикер; для текста — ввод текста; и т.д.).

## Реализация (набросок)

- В `StreamLayersOverlay` заменить кнопку на «Add source» + `DropdownMenu`/под-лист с пунктами типов.
- Пока реальны только «Картинка» (SAF) и тех. «Тест-оверлей» (debug) — остальные пункты добавляются
  по мере реализации соответствующих типов слоёв.
- Тривиально и обратимо (чисто UI над уже существующими методами добавления слоёв).
