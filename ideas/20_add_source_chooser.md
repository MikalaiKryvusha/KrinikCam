# Idea 20 — Кнопка «Добавить источник» с выбором типа (а не «добавить картинку»)

> Источник: Криник, чат 2026-06-29 (в ходе работы над мульти-источниками, Idea 19).
> Статус: 🟡 запланировано (часть дорожной карты мульти-источников).

## Суть

В панели «Scene layers» кнопка добавления слоя сейчас называется «Add image overlay» — это
неправильно по смыслу. Кнопка должна быть **общей: «Добавить источник»**. По нажатию открывается
выбор КОНКРЕТНОГО типа источника, и картинка — лишь один из вариантов.

## Как должно быть

- Кнопка **«Добавить источник» / «Add source»** (одна, общая).
- Тап → меню/лист выбора типа источника. Состав по приоритету Q2 (interview_005):
  - 🖼 **Картинка (PNG/JPG)** — есть (SAF-пик, Idea 19 фаза 1).
  - 🎞 **Видео-оверлей** (в т.ч. анимации-стикеры с альфой) — `SurfaceFilterRender`.
  - 🔤 **Текст** — `TextObjectFilterRender`.
  - 📷 **Вторая камера** (встроенная фронталка) — Camera2 → `SurfaceFilterRender`.
  - 🟥 **Заглушка «ЗАГЛУШКА KrinikCam»** — особый fallback-источник (Q4) — фаза 2 дорожной карты.
  - 🌐 **Браузер** (`AndroidViewFilterRender` + WebView) — на потом.
  - 🖥 **Захват экрана** — минимальный приоритет, на будущее.
- По мере реализации типов — просто добавляются пункты в это меню. Каждый тип внутри ведёт к своему
  потоку настройки (для картинки — SAF-пикер; для текста — ввод текста; и т.д.).

## Реализация (набросок)

- В `StreamLayersOverlay` заменить кнопку на «Add source» + `DropdownMenu`/под-лист с пунктами типов.
- Пока реальны только «Картинка» (SAF) и тех. «Тест-оверлей» (debug) — остальные пункты добавляются
  по мере реализации соответствующих типов слоёв.
- Тривиально и обратимо (чисто UI над уже существующими методами добавления слоёв).
