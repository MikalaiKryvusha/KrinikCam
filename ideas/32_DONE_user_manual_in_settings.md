# Idea 32 — User Manual (Пользовательское руководство) в Настройках

> **Статус:** ✅ ПРИНЯТО Криником (2026-07-06), реализация — `plans/06_user_manual.md`.
> Источник: заметки Криника (чат + этот файл, 2026-07-06).

## Суть
На экран **Settings** добавить кнопку, открывающую экран **User Manual** — встроенное руководство,
которое живёт ВНУТРИ приложения и описывает устройство и функционал KrinikCam. Ведётся
СОПРОВОДИТЕЛЬНО: развиваем/переделываем фичу → синхронно правим соответствующий раздел руководства.

## Зачем (ценность)
- KrinikCam = «мобильный OBS»: функций много и они растут. Без руководства неопытный пользователь
  теряется, прошаренный — не понимает «магию под капотом».
- Руководство В приложении (а не только в README/на сайте) — всегда под рукой, оффлайн, версия ровно
  та, что установлена.

## Два уровня читателя (оба в одном разделе, по оглавлению)
1. **Новичок** — «как этим пользоваться»: кратко, наглядно, по шагам, при нужде с картинками. Быстрый
   поиск по оглавлению → нашёл свою фичу → понял, как она устроена в KrinikCam и какие нюансы.
2. **Прошаренный** — «как это работает под капотом»: ключевые архитектурные решения БЕЗ сильной
   детализации. Напр. как работает **композитор** (собирает итоговый кадр из слоёв), какие библиотеки
   по каким потокам данных какие типы передают. Это НЕ конспект кода — это разъяснение сложных
   технических моментов, дающее понимание, как работает такое сложное приложение.

## Требования
- **Локализация в будущем:** закладываемся сразу (тексты — ресурсы/структура под перевод), но первый
  заход можно на одном языке.
- **Оформление:** красивая, лаконичная, структурная разметка (по аналогии с хорошо отрендеренным
  Markdown — заголовки, списки, выделения); наглядно, полезно, познавательно.
- **Каркас сейчас, детали потом:** сначала скелет с оглавлением и ключевыми архитектурными тезисами;
  глубокую детализацию наращиваем по мере развития фич.

## Ключевая конвенция (внесена в KAIF-канон)
Развивая/переделывая фичу — **высокоуровнево описываем её в разделе User Manual** и держим раздел в
соответствии с текущим прогрессом. Тезисно, но всегда есть. См. `AGENT_GUIDE.md` → «User Manual».
