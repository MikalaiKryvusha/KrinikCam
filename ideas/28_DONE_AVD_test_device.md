Нужно разработать навык / команды для запуска тестирования на Android Virtual Device.

Android Studio позволяет эмелировать AVD, запускать там Андроид и тестовые приложения. Нужно разработать под это все необходимые инструменты и бысть способным, при необходимости, когда реальный физический девайс Криника недоступен - уметь работать с виртуальным девайсом и дебажить во время разработки приложение на виртуальном устройстве.

В виртуальных устройствах есть свои виртуальные камеры - наше приложение будет ДУМАТЬ что это реальная физическая камера устройства - можно тестировать и дебажить словно с реальной камерой устрйоства.
## ✅ СТАТУС: РЕАЛИЗОВАНА (2026-07-18, ночной цикл, tools/avd.mjs) — DONE

`node tools/avd.mjs create|start|stop|status|smoke` — эмулятор как тест-девайс агента:
- Использует ГОТОВЫЙ AVD Android Studio (дефолт Pixel_7_Pro_Android_15, API 35; override KCAM_AVD).
  Собственное создание через легаси avdmanager невозможно на JBR 17 (javax.xml.bind удалён в 11+,
  поймано живьём) — создание новых AVD остаётся за студией, скрипт их подхватывает.
- start: headless (-no-window, swiftshader), ждёт sys.boot_completed; чистит СТЕЙЛ-ЛОКИ убитого
  инстанса (иначе «Running multiple emulators with the same AVD»); adb строго с -s (рядом планшет).
- smoke: полный харнес-цикл на эмуляторе (ADB_DEVICE=emulator-5554) с функциональным порогом
  `--min-fps 3` (софт-GPU рендерит ~5 fps — проверяем КОРРЕКТНОСТЬ пайплайна, не скорость;
  параметр --min-fps добавлен в smoke.mjs, дефолт на физике остался строгим 15).

**Приёмка наблюдением:** эмулятор поднят headless → APK установлен → SMOKE PASS (54 кадра/10с,
запись 1920×1080 валидна, CMD-контракт зелёный) → контрольный smoke на ПЛАНШЕТЕ со строгим
порогом тоже PASS (не сломан). Эмулированные камеры AVD доступны как Camera2 (select-source builtin).
