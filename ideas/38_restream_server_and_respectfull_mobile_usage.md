Я гуглил и узнал следующее:

- Одновременное кодирование в разрых кодеках на одном SoC - это не во всех SoC возможно и это дополнительно требудет ресурсов, что сильнее выедает АКБ мобильного устройства. Из этого следует, что большого смысла делать одновременную поддержку работы сразу нескольких кодеков - нецелесообразно. Логичнее будет сфокусироваться на кодировке одним кодеком для мультистриминга.

- Однако, кодировка одним кодеком для мультистриминга имеет свои ограничения - если хочется хорошего качества одновременно в N стриминговых сервисах, то это значит, что нужен канал пропускания мобильной сети равный битрейту кодека умноженному на число стриминг сервсиов - может быть очень много мегабит, и может быть сильный треуд офф между качеством стрима и числом платформ.

- Из второго пункта следует, что нужно вот как сделать: KrinikCam может для мультистрима работать в двух режимах
    - самостоятельный - сама рассылает потоки во все настроенные стриминг платформы
    - с поморью Self-Hosted ReStream Server - это будет отдельное ПО, запущенное на личном ПК в Docker. Если KrinikCam настроена на работу с этим сервером, то вот как это работает: KrinikCam понимает, что все настроенные сервисы для мультистрима нужно передать Self-Hosted ReStream Server, чтобы он настроил стриминги в эти серверы так, как KrinikCam запросил, а сам KrinikCam при запуске стриминга передаёт поток только одним каналом самому Self-Hosted ReStream Server, а уже Self-Hosted ReStream Server согласно настройкам стриминг платформ - распределяет получаемый от KrinikCam поток на все N стриминговые платформы. Как правило у ПК с развёрнутым Self-Hosted ReStream Server будет значительно более широкий канал интернета, так что он может легко получить входной поток от KrinikCam, и его размножить на N выходных потоков для соответствующих платформ, настроенных в KrinikCam. Такой подходи позволит стримить через слабую мобильную сеть клиентского устройства с KrinikCam в относительно высоком качесве сразу на несколько платформ. И все это работает в рамках одной экосистемы KrinikCam, без сторонних платных сервисов.
