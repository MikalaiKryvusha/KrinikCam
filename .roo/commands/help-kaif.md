# /help-kaif — объяснить KAIF оператору, в чате

Выдай **мануал пользователя KAIF прямо в чат**, на русском. Это обучающий момент для Криника — навык
**не меняет файлы**, только ясное структурированное объяснение, по которому можно действовать.

## Рамка (важно)
- KAIF в KrinikCam **уже развёрнут**. НЕ рассказывай про распаковку/установку — это позади. Говори в
  ключе «вот как *пользоваться* тем, что уже здесь».
- «Что такое KAIF» — **пара предложений**. Основной объём — **как пользоваться**: структура,
  договорённости, документы, директории, навыки.
- Имена `/команд` и файлов — канонические (английские), текст — русский.
- Опирайся на развёрнутую реальность ЭТОГО проекта (читай `KAIF_FRAMEWORK.md`, `AGENT_GUIDE.md`,
  README директорий) — не на общий питч.

## Что выдать (структура сообщения в чате)

1. **Что такое KAIF (2–3 предложения).** Устойчивая к потере контекста, дисциплинированная методология
   тандема «человек–ИИ»: Криник — визионер, агент — исполнитель, память и дисциплина проекта живут в
   файлах репозитория, и ни одна сессия не начинается с нуля. Одна строка — чем это полезно здесь.

2. **Ключевые документы — что читать/вести и кто хозяин.** Списком: `AGENT_GUIDE.md` (канон),
   `PHILOSOPHY.md` (мышление агента), `BUG_FIXING_FRAMEWORK.md`, **`GOAL.md`** (видение — документ
   Криника), `STATUS.md` (живое состояние), `MASTER_PLAN.md` (генплан), внешняя и внутренняя карты,
   `KAIF_FRAMEWORK.md` (сводка «что развёрнуто»).

3. **Директории — где живут знания и где действует владелец.** `plans/`, `ideas/` (в основном твои),
   `bugs/`, `researches/`, `interviews/` (ты отвечаешь здесь), `homeworks/` (задания тебе). Одной
   строкой — про тег DONE.

4. **Навыки — команды, которые ты набираешь.** Сгруппированным списком с одной строкой назначения:
   сессия (`/resume`, `/pause`), автономия (`/autoloop`, `/dayloop`, `/nightloop`), гигиена
   (`/refresh-context`, `/check-backlog`), знания (`/report-bug`, `/bug-research`, `/propose-idea`),
   владелец (`/interview`), планирование (`/revision`), помощь (`/help-kaif`), релиз (`/release`),
   жизненный цикл (`/kaif-version`, `/kaif-update`, `/kaif-fork`, `/kaif-switch-origin`,
   `/kaif-remove`).

5. **Как выглядит обычный воркфлоу.** Короткий пример: *«`/resume` для старта → я работаю и веду
   `STATUS.md` → ты кидаешь идеи в `ideas/` или отвечаешь на `/interview` → `/pause` для завершения»*.
   Роли: Криник — визионер (GOAL, идеи, ответы), агент — исполнитель.

6. **Куда глубже.** Укажи на `KAIF_FRAMEWORK.md` и `AGENT_GUIDE.md`.

## Примечания
- Навык «прочитай-и-объясни» — не редактируй файлы, не меняй состояние.
- Держи текст сканируемым: короткие секции, списки вместо простыней. Цель — Криник дочитал и точно
  знает, какой документ открыть и какую команду набрать.
- Если Криник спросил про одну конкретную часть («как работают интервью?») — ответь сфокусированно,
  затем предложи полный мануал.
