# ДЗ 05 — Проверить мультистрим (ютуб + инстаграм разом)

> Для Криника. Заведено: 2026-07-07 (дейлуп). Связано: plans/07_multistream_mvp.md.

## Что сделано (движок готов)

Движок мультистрима реализован и проверён смоуком:
- **S1** — стриминг переведён на `MultiStream` (RootEncoder): один энкодер → до 4 RTMP-выходов.
- **S3** — `startStream(List<профилей>)` стартует каждую платформу на своём выходе (ютуб=0, инстаграм=1…),
  кодирование ОДИН раз. Одно-выходной путь сохранён (smoke PASS).

Осталось UI мультивыбора платформ (S2/S4) + **живая сверка на реальных платформах — это ДЗ**.

## Что нужно от тебя

Чтобы подтвердить мультистрим вживую, нужен один из вариантов:

### Вариант A (autonomous) — разрешить локальный RTMP-полигон
Разово одобрить скачивание бинаря MediaMTX для локального RTMP-сервера:
```
node tools/rtmp-server.mjs start
```
Тогда я смогу сам протестировать 2 одновременных выхода (оба на локальный сервер, разные ключи) без
реальных платформ и без тебя. **Предпочтительно** — разблокирует автономную проверку.

### Вариант B — реальные ключи (финальная сверка)
Дать stream-ключи:
- **YouTube:** Studio → «Прямые трансляции» → RTMP-URL (`rtmp://a.rtmp.youtube.com/live2`) + ключ потока.
- **Instagram:** Live RTMP-URL + ключ (нюанс: IG Live RTMP иногда требует Business/Producer-доступа —
  если ключа нет, проверим на ютубе + локальном сервере как второй выход).

Куда вписать: в приложении Настройки → Платформы → профиль (URL + ключ). Для мультистрима нужен UI
мультивыбора (S4) — доделаю; пока можно проверить один профиль на реальном ключе + второй на локальном.
