# ДЗ 06 — Спрятать release-ключ подписи KrinikCam (interview_009 Q3=A)

> Создано: 2026-07-12. Приоритет: сделать в течение недели (пока ключ существует в одном экземпляре
> на этом маке). ~10 минут.

## Что произошло

Я сгенерировал **настоящий ключ подписи release-сборок** (bugs/37 №2). Теперь все release-APK будут
подписаны им — обновления будут вставать поверх старых версий с любой машины сборки, где есть этот
ключ.

## Почему это важно (одной строкой)

**Потеря этих двух файлов = невозможность выпускать обновления KrinikCam навсегда** (Android не
поставит апдейт с другой подписью — только удаление приложения со всеми настройками у каждого
пользователя).

## Что спрятать (два файла с этого мака)

| # | Файл | Что это |
|---|------|---------|
| 1 | `~/keystores/krinikcam.keystore` | сам ключ (вне репозитория; в git не попадает: `*.keystore` в .gitignore) |
| 2 | `~/ai_sandbox/KrinikCam/keystore.properties` | пароль к ключу (в корне репо, но **gitignored** — в GitHub не уходит) |

## Куда спрятать (сделай ОБА пункта — два независимых места)

1. **Менеджер паролей** (1Password/Bitwarden/связка iCloud): создай запись «KrinikCam release
   keystore», приложи файл `krinikcam.keystore` как вложение и скопируй в заметку полное содержимое
   `keystore.properties` (там пароль).
2. **Личное облако** (Google Drive / iCloud Drive), папка типа `Backups/KrinikCam/`: положи копию
   `krinikcam.keystore` и `keystore.properties`. Папку не расшаривать.

Проверка: файл `.keystore` открылся из облака заново (размер ~5 КБ, не 0) — готово.

## Чего НЕ делать

- НЕ клади эти файлы в репозиторий/GitHub (даже приватный), в Telegram «Избранное», в почту.
- НЕ переименовывай и не «чисти» `~/keystores/` при уборке диска.
