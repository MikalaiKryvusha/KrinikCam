# /autoloop — длительная автономная работа серией над пулом задач

Криник уходит / не хочет вмешиваться. Задача агента: **войти в самостоятельный цикл и грайндить пул
АВТОНОМНЫХ задач** одну за другой, пока пул не исчерпан или не появится блокер. Между задачами —
короткие отчёты в чат, чтобы Криник мог заглянуть и увидеть прогресс.

## 🔁 САМОВЫЗОВ и самоперезапуск (агент запускает себя сам)

Агент имеет право и ДОЛЖЕН вызывать этот навык **по собственной инициативе** (через Skill-инструмент),
не дожидаясь явной команды Криника, когда выполнены ВСЕ условия:
- есть незакрытые задачи в АВТОНОМНОМ пуле (`STATUS.md` → «🤖 АВТОНОМНЫЙ ПУЛ ЗАДАЧ»);
- нет активной интерактивной задачи/вопроса от Криника прямо сейчас (он отошёл / дал автономию);
- устройство доступно (`adb devices` = `device`) и сборка рабочая.

Кроме того, **цикл самоперезапускается**: завершив одну задачу (коммит + короткий отчёт), агент САМ
переходит к следующей итерации этого же навыка — берёт следующую задачу из пула, без ожидания
команды. Так серия идёт длительно и непрерывно. Останавка — только по условиям из раздела «Когда
ОСТАНОВИТЬ цикл» ниже (пул исчерпан / нужен Криник или камера / деструктив / развилка на интервью).

> Не злоупотребляй: самовызов — для автономного пула. Если Криник дал конкретную интерактивную
> задачу — выполняй её, а не уходи в луп. Если он явно сказал «стоп / хватит / пауза» — останавливайся.

> 🧠 Работай по `PHILOSOPHY.md` (ПРОСТО, KISS, Оккам) и `BUG_FIXING_FRAMEWORK.md` (3 попытки → стоп →
> `/bug-research`). Активно комментируй код (правило проекта). Пиши в чат естественным языком о том,
> что делаешь.

## ЖЕЛЕЗНОЕ ограничение автономности

**Камера НЕ подключена. Криника нет.** Поэтому НЕЛЬЗЯ брать задачи, требующие:
- видеопотока с USB-камеры, подключения/отключения камеры;
- запуска RTMP-стрима / проверки на YouTube;
- любого участия Криника (его рук, глаз, решений по UI/бренду/архитектуре).

Если задача (или её проверка) упирается в это — **НЕ бери её, отложи, возьми другую.** Когда камера
не подключена, приложение показывает `StandbyPlaceholder` — весь UI (Settings, Platforms overlay,
FAB, диалоги, навигация, dev-меню) доступен для автономного теста через `ui.mjs`.

## Шаг 0. Подготовка (один раз в начале)

1. Прочитай: `STATUS.md` (секция «🤖 АВТОНОМНЫЙ ПУЛ ЗАДАЧ»), `PHILOSOPHY.md`, `AGENT_GUIDE.md`,
   `BUG_FIXING_FRAMEWORK.md`, нужные `plans/ideas/*` и `bugs/*`.
2. Проверь окружение:
   - `adb devices` → должен быть `device` (192.168.1.3:5555). Если `offline` — `adb connect`.
   - `export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"` для сборок.
3. Собери/освежи рабочий список пула из STATUS. Сообщи Кринику одним абзацем: какие задачи в пуле,
   с какой начинаешь и почему.

## Цикл по КАЖДОЙ задаче (повторяй для серии)

1. **Выбор** — возьми следующую автономную задачу из пула (приоритет из STATUS; рекоменд. порядок:
   Idea 07 dev-меню → Idea 01 import/export → Idea 05 SEO → UI-полировка). Проверь, что её можно
   проверить БЕЗ камеры/стрима/Криника. Если нет — отложи, бери следующую.
2. **Понять задачу просто** (PHILOSOPHY) — сформулируй в 1-2 предложениях. Если для багов — заведи/
   открой документ в `bugs/`.
3. **Реализуй** точечно, с комментариями. Не усложняй.
4. **Собери**: `node tools/build.mjs --no-ui` (с JAVA_HOME). Если `e:` ошибки — чини, не коммить
   сломанное.
5. **Поставь и запусти**: `node tools/adb.mjs install` → `node tools/ui.mjs restart debug`.
6. **Проверь автономно** через `ui.mjs`: `dump` (DOM+координаты), `tap`/`swipe`/`find`, `screen`
   (скриншот для визуальной сверки — ВНИМАТЕЛЬНО смотри на результат, не выдавай желаемое за
   действительное), `allow` (системные диалоги), `orient` (ориентация). Камеру НЕ трогай.
7. **Цикл фикса** при баге: фикс → билд → тест → логи (`adb logcat`, свежие по времени). Правило
   **3 попыток** → `/bug-research` (без кода) → потом фикс.
8. **Зафиксируй знания**: для багов — рефлексия в `bugs/NN_*.md`; для фич — статус/дата в
   `plans/ideas/*`; обнови `STATUS.md`.
9. **Закоммить** маленьким коммитом (прогресс не теряем):
   `node tools/commit.mjs "<тип>: <что сделано>"` (стиль из AGENT_GUIDE, с Co-Authored-By).
10. **Короткий отчёт в чат** (1-3 строки): что сделал, что проверил, что дальше. → к следующей задаче.

## Самопейсинг (чтобы цикл был ДЛИТЕЛЬНЫМ)

- Иди задача за задачей без остановок на подтверждения (если задача не деструктивна).
- Если ждёшь фоновую операцию (длинный билд) — продолжай по готовности; не зови Криника зря.
- Если нужно «продолжать сам по таймеру» — используй механизм `/loop` (ScheduleWakeup) с разумным
  интервалом, передавая обратно этот же навык, чтобы возобновлять цикл.

## Когда ОСТАНОВИТЬ цикл (и доложить Кринику)

- Пул автономных задач исчерпан (все камера-зависимые/требующие Криника отложены).
- Все оставшиеся задачи требуют камеру/стрим/Криника.
- Серьёзная развилка UI/UX/бренд/архитектуры, которую агент НЕ должен решать сам → оформи интервью
  (`/interview`) и поставь на паузу.
- Что-то деструктивное/необратимое (релиз, удаление, force-push) — НЕ делай сам, спроси.

В конце цикла — итог: что сделано за серию (список коммитов), что отложено и почему, что предложить
дальше. Коммиты по ходу гарантируют, что прогресс не потерян.

## Заметки

- НЕ публикуй релизы (`release.mjs`), НЕ делай force-push, НЕ удаляй чужое — это вне автономии.
- Держи дерево чистым перед коммитом; `*.apk`/`README.pdf` gitignored.
- Скриншоты — в `tools/screenshots/` (gitignored). Логи читай только СВЕЖИЕ (сверяй по времени).
- Если камеру вдруг подключат и появится задача под неё — это уже не автономный режим, доложи Кринику.
