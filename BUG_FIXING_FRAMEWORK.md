## UI Automation Tool — инструмент для тестирования

**Инструмент создан: `tools/ui.mjs`**
**Полное руководство: `tools/UI_AUTOMATION_GUIDE.md`**

Никогда не угадывай координаты из скриншотов. Всегда используй:

```bash
# Что на экране?
node tools/ui.mjs dump

# Тапнуть кнопку по тексту (точные координаты из DOM)
node tools/ui.mjs tap "go live"
node tools/ui.mjs tap "menu"

# Проверить наличие элемента
node tools/ui.mjs find "platforms"
```

**Как это работает:**
1. `adb shell uiautomator dump` → XML с `bounds="[x1,y1][x2,y2]"` для каждого элемента
2. Парсится `text`, `content-desc`, `resource-id` — матч по частичному совпадению
3. Вычисляется центр `((x1+x2)/2, (y1+y2)/2)` → `adb shell input tap`

**Типичный тест-цикл баг-фиксинга:**
```bash
# 1. Запустить приложение
adb -s 192.168.1.3:5555 shell am start -n com.kriniks.kcam.debug/com.kriniks.kcam.MainActivity
sleep 3

# 2. Открыть меню и нажать Go Live
node tools/ui.mjs tap "menu" && sleep 1.5 && node tools/ui.mjs tap "go live"
sleep 2

# 3. Сразу читать логи
adb -s 192.168.1.3:5555 pull /sdcard/Android/data/com.kriniks.kcam.debug/files/logs/kcam_$(date +%Y-%m-%d).log /tmp/kcam_today.log
grep -E "startStream|RTMP|Broken|blocked|crash|error" /tmp/kcam_today.log | tail -40

# 4. Смотреть крэш-файлы если приложение упало
adb -s 192.168.1.3:5555 shell ls /sdcard/Android/data/com.kriniks.kcam.debug/files/crash_*.txt 2>/dev/null
```

**ВАЖНО при работе с логами:**
- Всегда проверяй текущее время (`date`) и время последней записи в лог-файле
- Не читай вчерашние логи — используй свежую дату
- Формат пути: `/sdcard/Android/data/com.kriniks.kcam.debug/files/logs/kcam_YYYY-MM-DD.log`
